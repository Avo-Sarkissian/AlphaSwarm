```python
async def _auto_trigger_advisory(app: FastAPI, cycle_id: str) -> None:
    """POST /api/advisory/{cycle_id} in-process after simulation completes.

    Uses httpx.ASGITransport to call the FastAPI ASGI app directly — no
    network socket, no port assumption. All route guards (409, 503) are
    re-used. Non-202 responses are swallowed: 409 means another task is
    already running (fine), 503 means portfolio unavailable (advisory
    impossible — expected degraded mode). Any network-level exception is
    logged and swallowed so it never surfaces as an unhandled exception.
    """
    try:
        transport = httpx.ASGITransport(app=app)  # type: ignore[arg-type]
        async with httpx.AsyncClient(transport=transport, base_url="http://testserver") as client:
            resp = await client.post(f"/api/advisory/{cycle_id}")
        if resp.status_code == 202:
            log.info("auto_advisory_trigger_accepted", cycle_id=cycle_id)
        elif resp.status_code == 409:
            log.info("auto_advisory_trigger_skipped_conflict", cycle_id=cycle_id)
        elif resp.status_code == 503:
            log.warning("auto_advisory_trigger_skipped_unavailable", cycle_id=cycle_id)
        else:
            log.warning(
                "auto_advisory_trigger_unexpected_status",
                cycle_id=cycle_id,
                status=resp.status_code,
            )
    except Exception:
        log.exception("auto_advisory_trigger_failed", cycle_id=cycle_id)
```

| Boundary | Description |
|----------|-------------|
| SimulationManager → FastAPI route | Auto-trigger POSTs to own endpoint in-process via ASGITransport — no external socket |
| `_auto_trigger_advisory` → advisory route guard chain | Route re-validates phase, portfolio presence, and concurrent task conflicts |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-999.1-01 | Denial of Service | `_on_task_done` → `asyncio.create_task` | mitigate | Task is named (`auto_advisory_{cycle_id}`); if loop is under pressure, task is queued not dropped; `_auto_trigger_advisory` has a top-level try/except so an unhandled exception in the coroutine cannot take down the event loop |
| T-999.1-02 | Information Disclosure | `_auto_trigger_advisory` logs | mitigate | Log only `cycle_id` (a non-PII UUID); never log portfolio data or advisory content — matches ISOL-04 pattern already in codebase |
| T-999.1-03 | Elevation of Privilege | `on_complete` callback bypasses HTTP auth | accept | There is no HTTP auth on the advisory route today (internal-only service); ASGITransport calls the route's existing guard chain (phase check, portfolio check, concurrency check) — no privilege escalation |
| T-999.1-04 | Spoofing | Duplicate trigger (two `_on_task_done` calls) | accept | `asyncio.Lock` in `SimulationManager` guarantees exactly one simulation task at a time; `_last_cycle_id` is reset to None immediately after reading; duplicate fires structurally impossible |
| T-999.1-05 | Tampering | `cycle_id` passed to POST URL | mitigate | `_validate_cycle_id` regex guard in advisory route rejects malformed IDs; `cycle_id` originates from `run_simulation()` return value (internal, not user input) — low risk but guard present |
</threat_model>
