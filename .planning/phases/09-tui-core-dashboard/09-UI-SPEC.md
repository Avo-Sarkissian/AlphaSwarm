# Phase 9 -- UI Design Contract

> Visual and interaction contract for the TUI Core Dashboard. Generated by gsd-ui-researcher. Adapted for Textual (Python terminal UI) -- not a web application.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Textual (>=8.1.1) |
| Preset | Custom dark theme via `textual.theme.Theme` |
| Component library | Textual built-in widgets (`Static`, `Container`, `Widget`) |
| Icon library | Unicode symbols only (no icon library) |
| Font | Terminal monospace (user's terminal font, no override) |

**Note:** This is a Python terminal UI, not a web application. shadcn, Tailwind, and browser-based design systems do not apply. All styling uses Textual CSS (TCSS) and the Textual theme variable system.

---

## Spacing Scale

Terminal UIs operate in character cells (1 unit = 1 character width or 1 line height). All spacing values are in cell units, not pixels.

| Token | Value | Usage |
|-------|-------|-------|
| none | 0 | Cell-to-cell adjacency in agent grid |
| xs | 1 | Inline padding within header labels, cell internal margin |
| sm | 2 | Gap between header sections (pipe-separated) |
| md | 3 | Padding around the grid container |
| lg | 4 | Margin between header bar and grid |

Exceptions: Agent grid cells are exactly 3 characters wide by 1 line tall. No fractional cell spacing.

---

## Typography

Terminal UI is monospace. "Size" refers to visual emphasis via weight and style, not pixel size. Textual renders in the user's terminal font at the terminal's configured size.

| Role | Style | Weight | Usage |
|------|-------|--------|-------|
| Body | normal | normal (400) | Agent cell content (currently empty -- cells are color-only) |
| Label | normal | normal (400) | Header status text, round counter, elapsed time |
| Heading | normal | bold (700) | App title "AlphaSwarm" in header |
| Status indicator | normal | bold (700) | Active simulation phase label (e.g., "Seeding", "Round 1") |

Only 2 weights used: **normal** (400) for data labels and **bold** (700) for emphasis (app title, active status). No italic, no underline in Phase 9.

### 60/30/10 Surface Distribution

| Role | Percentage | Value | Usage |
|------|------------|-------|-------|
| Dominant (60%) | `#121212` | Terminal background, empty space around grid |
| Secondary (30%) | `#1E1E1E` | Header bar background, grid container surface |
| Accent (10%) | `#4FC3F7` | Status indicator dot, app title text |

Accent reserved for: app title in header, active status indicator dot, active round number. Never for grid cells (grid uses semantic signal colors below).

### Agent Grid Signal Colors

These are the primary visual language of the dashboard. Each cell encodes signal via hue and confidence via brightness (lightness channel in HSL).

| Signal | Base Hue (HSL) | Min Lightness | Max Lightness | Example Range |
|--------|---------------|---------------|---------------|---------------|
| BUY (bullish) | H=120, S=60% | L=20% (conf=0.0) | L=50% (conf=1.0) | `hsl(120,60%,20%)` to `hsl(120,60%,50%)` |
| SELL (bearish) | H=0, S=70% | L=20% (conf=0.0) | L=50% (conf=1.0) | `hsl(0,70%,20%)` to `hsl(0,70%,50%)` |
| HOLD | -- | -- | -- | `#555555` (fixed, no brightness scaling) |
| PENDING (no decision) | -- | -- | -- | `#333333` (fixed dim gray per D-05) |

**Brightness formula:** `lightness = 20 + (confidence * 30)` where confidence is 0.0--1.0. This maps:
- confidence 0.0 -> lightness 20% (very dim, barely visible)
- confidence 0.2 -> lightness 26% (dim but visible -- minimum legibility floor)
- confidence 0.5 -> lightness 35% (moderate)
- confidence 1.0 -> lightness 50% (full intensity)

**Minimum brightness floor:** Lightness never goes below 20%. Even confidence=0.0 produces a visible cell against the `#121212` background.

## Component Inventory

### AgentCell (Custom Widget)

- Dimensions: 3 characters wide, 1 line tall
- Content: No text content (empty `Static` or custom `Widget`). Color-only display.
- Background: Computed from signal + confidence per color table above
- Border: None
- States: PENDING (gray `#333333`), BUY (green gradient), SELL (red gradient), HOLD (gray `#555555`)
- Interaction: None in Phase 9 (no click, no hover -- deferred)

### AgentGrid (Container)

- Layout: 10 columns x 10 rows, no gaps between cells
- Textual CSS: `grid-size: 10 10;` with `grid-gutter: 0;`
- Dimensions: 30 characters wide (10 cells * 3 chars) x 10 lines tall
- Background: `$surface` (`#1E1E1E`)
- Padding: 3 cells horizontal, 1 line vertical (centered in terminal)
- Agent mapping: Sequential row-by-row. Agent index 0 -> cell [0,0], agent index 99 -> cell [9,9] (per D-03)

### HeaderBar (Container/Static)

- Layout: Single horizontal line, full terminal width
- Background: `$surface` (`#1E1E1E`)
- Height: 1 line
- Content format: `  AlphaSwarm  |  Round X/3  |  [dot] Status  |  HH:MM:SS  `
- Separator: ` | ` (pipe with spaces) in `$secondary` color (`#78909C`)
- App title: "AlphaSwarm" in bold, `$primary` color (`#4FC3F7`)
- Round counter: "Round X/3" in normal weight, `$foreground` (`#E0E0E0`)
- Status indicator: Unicode bullet `\u25CF` in `$accent` (`#4FC3F7`) followed by `SimulationPhase` display label in bold `$foreground`
- Elapsed time: "HH:MM:SS" in normal weight, `$foreground`

**SimulationPhase display labels:**

| Enum Value | Display Label |
|------------|--------------|
| `IDLE` | Idle |
| `SEEDING` | Seeding |
| `ROUND_1` | Round 1 |
| `ROUND_2` | Round 2 |
| `ROUND_3` | Round 3 |
| `COMPLETE` | Complete |

## Interaction Contract

### Snapshot-Based Rendering (TUI-02)

- Timer: `set_interval(1/5)` -- fires every 200ms
- On tick: Read `StateStore.snapshot()` for immutable `StateSnapshot`
- Diff: Compare current snapshot agent states against previous snapshot
- Update: Only call `cell.refresh()` on cells whose signal or confidence changed
- No full-screen redraws. Changed-cells-only rendering.

### Simulation Lifecycle (D-01)

- TUI app owns the event loop
- Simulation runs as a `Textual Worker` (background asyncio task)
- StateStore is the shared bridge: simulation writes, TUI reads
- No IPC, no separate processes

### User Input

- **Keyboard:** `q` or `Ctrl+C` to quit (standard Textual binding)
- **Mouse:** None in Phase 9 (no clickable elements)
- **Resize:** Textual handles terminal resize natively. Grid remains centered.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `python -m alphaswarm tui "rumor text"` (CLI invocation, no in-app CTA button) |
| Idle state heading | Displayed in header as "Idle" -- grid shows 100 pending gray cells |
| Idle state body | No separate empty state message. The 100 dim gray cells ARE the empty state. |
| Error state (simulation crash) | Simulation Worker posts error via Textual `notify()`: "Simulation failed: {error}. Press q to exit." |
| Error state (Ollama not running) | Startup error before TUI renders: "Error: Ollama is not running. Start Ollama and retry." (printed to stderr, app exits) |
| Error state (Neo4j not running) | Startup error before TUI renders: "Error: Neo4j is not reachable. Start the container and retry." (printed to stderr, app exits) |
| Destructive confirmation | None -- Phase 9 has no destructive actions. Quit is immediate (no confirmation dialog). |
| Complete state | Header shows "Complete". Grid freezes at final Round 3 state. No automatic exit. |

## Accessibility Notes

- Color is the primary encoding. Phase 10 may add text overlays for accessibility. Phase 9 is color-only per D-04.
- Terminal screen readers can read the header bar text (status, round, elapsed time).
- High contrast between grid cells (`L>=20%`) and background (`#121212`, `L~7%`) ensures minimum 3:1 luminance ratio.
- HOLD (`#555555`, `L~33%`) and PENDING (`#333333`, `L~20%`) are visually distinct from each other and from BUY/SELL hues.
