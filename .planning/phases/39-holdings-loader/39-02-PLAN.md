<objective>
Close HOLD-03: expose `GET /api/holdings` as the ONLY web route permitted to import `alphaswarm.holdings`, backed by an eager-loaded `PortfolioSnapshot` on `app.state.portfolio_snapshot`. This plan produces the configuration field (D-05), the route module (D-06), the lifespan wiring (D-07, D-08), and the integration tests that prove all three behaviors.

The route implementation is deliberately minimal: it reads `request.app.state.portfolio_snapshot` and serializes through pydantic `BaseModel` classes with `Decimal`-as-string fields (Pitfall 1 — never `float()` a `Decimal`). When the lifespan loader failed, `app.state.portfolio_snapshot is None` and the handler returns 503 with a generic body — the raw `HoldingsLoadError` message and filesystem path are logged server-side but NEVER leak into the HTTP response (T-39-06 from Plan 01's threat model).

Three integration points land here: (1) `AppSettings.holdings_csv_path: Path` uses pydantic-settings' existing `env_prefix='ALPHASWARM_'` to expose `ALPHASWARM_HOLDINGS_CSV_PATH` with default `Path('Schwab/holdings.csv')` (D-05); (2) the FastAPI `lifespan()` in `web/app.py` wraps `HoldingsLoader.load(settings.holdings_csv_path)` in `try/except HoldingsLoadError` and stores either the snapshot or `None` on `app.state.portfolio_snapshot` (D-07, D-08); (3) `create_app()` adds one line: `app.include_router(holdings_router, prefix="/api")`.

Importlinter: `pyproject.toml` receives ZERO edits. `alphaswarm.web.routes.holdings` is already listed in `tests/invariants/test_importlinter_coverage.py::_KNOWN_NON_SOURCE` (Phase 37 pre-whitelisted it per D-04). Adding it to `source_modules` would BREAK `test_source_modules_does_not_list_whitelisted_packages`. The D-04 whitelist MEANS "this module can import `alphaswarm.holdings` and no other web module can" — which is exactly the HOLD-03 isolation contract.

Tests live under `tests/integration/` because FastAPI `TestClient` needs in-process sockets and `tests/integration/conftest.py::pytest_collection_modifyitems` auto-applies the `enable_socket` marker that overrides the global `--disable-socket` gate.         account_number_hash=snapshot.account_number_hash,
