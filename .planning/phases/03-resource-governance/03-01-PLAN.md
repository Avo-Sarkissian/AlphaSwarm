        async def start_monitoring(self) -> None: ...

    `ResourceGovernor` class:
    - Constructor takes `GovernorSettings` (not just baseline_parallel). Creates `TokenPool(settings.baseline_parallel)`, `MemoryMonitor(settings)`, initializes state to `RUNNING`.
    - `_state: GovernorState` -- current state machine state
    - `_active_count: int` -- currently checked-out slots
    - `_last_successful_inference: float` -- monotonic timestamp, updated on `release(success=True)` (D-08)
    - `_crisis_start: float | None` -- monotonic timestamp when crisis began
    - `_consecutive_green_checks: int` -- counter for scale-up eligibility (D-02)
    - `_adjustment_lock: asyncio.Lock` -- protects all slot count modifications (grow, shrink, limit changes)
    - `_monitor_task: asyncio.Task | None` -- background monitoring task
    - `_settings: GovernorSettings` -- stored reference

    Methods:
    - `async def acquire(self) -> None`: Check cached pressure (if CRISIS/PAUSED, wait on an asyncio.Event `_resume_event`). Then acquire token from pool. Increment `_active_count`.
    - `def release(self, *, success: bool = True) -> None`: Return token to pool. Decrement `_active_count`. If `success`, update `_last_successful_inference` to `time.monotonic()`. Per D-08, this resets crisis timeout.
    - `async def start_monitoring(self) -> None`: Start background `_monitor_loop` as an asyncio.Task.
    - `async def stop_monitoring(self) -> None`: Cancel the monitor task, await it.
    - `async def _monitor_loop(self) -> None`: Runs every `check_interval_seconds` (2s default). Reads `MemoryMonitor.read_combined()`. Applies state transitions per research Pattern 4 state machine. Logs state changes (D-10). On state change, updates `_resume_event` (set if RUNNING/THROTTLED/RECOVERING, clear if PAUSED/CRISIS).
    - `async def _apply_state_transition(self, reading: MemoryReading) -> None`: State machine logic:
      - In RUNNING: if `is_crisis` -> CRISIS (drop to 1 via shrink). If `is_pause_zone` -> PAUSED. If `is_throttle_zone` -> THROTTLED. If `is_scale_up_eligible` -> increment `_consecutive_green_checks`, if >= `scale_up_consecutive_checks` AND current_limit < max_parallel, grow by `slot_adjustment_step` (D-02).
      - In THROTTLED: if `is_crisis` -> CRISIS. If `is_pause_zone` -> PAUSED. If NOT `is_throttle_zone` -> RUNNING with gradual +2 recovery (D-01).
      - In PAUSED: if `is_crisis` -> CRISIS. If NOT `is_pause_zone` -> THROTTLED or RUNNING.
      - In CRISIS: if Green -> RECOVERING (reset to baseline per D-03). Check crisis timeout: if `time.monotonic() - _crisis_start > crisis_timeout_seconds` AND `time.monotonic() - _last_successful_inference > crisis_timeout_seconds` -> raise GovernorCrisisError (D-07).
      - In RECOVERING: gradual +2 per check until baseline reached (D-01), then -> RUNNING. If `is_crisis` again -> CRISIS.
    - `def report_wave_failures(self, success_count: int, failure_count: int) -> None`: Per D-04/D-05, if `failure_count / (success_count + failure_count) >= batch_failure_threshold_percent / 100`, shrink by `slot_adjustment_step`, minimum 1 slot. Log WARNING.
    - `is_paused` property: returns True if state is PAUSED or CRISIS.
    - `current_limit` property: delegates to `_pool.current_limit`.
    - `active_count` property: returns `_active_count`.
    - `state` property: returns `_state` (for observability).
    - `__aenter__` / `__aexit__`: preserved for backward compat, calls acquire/release.

    Logging per D-09/D-10:
    - `structlog.get_logger(component="governor")`
    - INFO for slot adjustments (grow/shrink/scale-up)
    - WARNING for throttle, pause, crisis entry
    - ERROR for abort (GovernorCrisisError)
