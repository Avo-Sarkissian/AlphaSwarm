    `GovernorState` enum: RUNNING, THROTTLED, PAUSED, CRISIS, RECOVERING (per research Pattern 4).

    `TokenPool` class (per research Pattern 1, **with debt tracking per review concern #1**):
    - `__init__(self, initial_size: int)` -- creates `asyncio.Queue[bool]` filled with `initial_size` True tokens. Initializes `_debt: int = 0`.
    - `async def acquire(self) -> None` -- `await self._pool.get()`
    - `def release(self) -> None` -- **Check debt first**: if `self._debt > 0`, decrement `_debt` and discard token (do NOT put back in queue). Otherwise, `self._pool.put_nowait(True)`. This is the debt pattern from the review: when `shrink()` cannot remove tokens because they are checked out, it records debt. When those tokens are eventually released, they are consumed by the debt instead of returning to the pool.
    - `def grow(self, amount: int) -> None` -- adds tokens, increments `_current_limit`
    - `def shrink(self, amount: int) -> int` -- **Two-phase shrink**: First, try to remove free tokens from queue via `try: self._pool.get_nowait()` (non-blocking). For each token that cannot be removed (QueueEmpty), add to `_debt` counter and still decrement `_current_limit`. Returns `amount` always (the limit change is immediate; actual token removal is deferred via debt). This avoids blocking and ensures `current_limit` accurately reflects the intended capacity.
