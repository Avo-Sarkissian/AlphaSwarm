# Phase 31 — UI Design Contract

> Visual and interaction contract for the Vue 3 SPA and force-directed agent graph. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (hand-rolled Vue 3 + D3 components) |
| Icon library | none (phase uses no icons beyond a single close X character) |
| Font | Inter (`Inter, -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`) |

Notes: This is a Python/Vue project, not a React/Next.js project. shadcn is not applicable. Components are hand-rolled Vue 3 SFCs with scoped CSS. D3 is the physics engine only; Vue owns the SVG DOM.

## Spacing Scale

Declared values (multiples of 4, 8-point base):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, node label offset from circle |
| sm | 8px | Compact element spacing, sidebar field gaps |
| md | 16px | Default element spacing, sidebar section padding |
| lg | 24px | Section padding, sidebar header/body separation |
| xl | 32px | Layout gaps between graph container and viewport edge |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions: Touch/click target for agent nodes is the SVG circle element itself (5-14px visible radius) plus a transparent hit area of 20px radius minimum to ensure comfortable click targeting on small nodes.

Note: SVG node radii (5-14px per bracket tier) are geometry parameters controlling the visual size of rendered circles, not layout spacing tokens. They are exempt from the 8-point spacing scale.

## Color

### Surface Colors

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0f1117` | Page background, graph SVG background |
| Secondary (30%) | `#1a1d27` | Sidebar background, status bar background |
| Accent (10%) | `#3b82f6` | See reserved-for list below |
| Destructive | `#ef4444` | SELL signal nodes only (dual-purpose: signal + destructive) |

Accent `#3b82f6` reserved for:
- Sidebar close button hover state
- Selected node ring (2px stroke around clicked agent node)
- "Waiting for Simulation" spinner/pulse animation
- Edge highlight on hover (future phase — not implemented in 31)

Note: `#ef4444` is intentionally shared between `--color-destructive` and `--color-signal-sell`. In Phase 32+, destructive confirmation UI uses `--color-destructive`; signal nodes use `--color-signal-sell`. Both resolve to the same hex value but are declared as separate tokens so they can diverge if needed in later phases.

## Component Inventory

### ForceGraph.vue

The hero component. Full-viewport SVG rendering 100 agent nodes with D3 force physics.

| Property | Value |
|----------|-------|
| Container | `<div>` filling available width (100vw minus sidebar when open, 100vw when closed) and 100vh |
| SVG | Fills container, `viewBox` auto-scaled to container dimensions |
| Node element | `<circle>` per agent, `r` = 5-14px by bracket tier |
| Node fill | Signal color (see Signal Colors table), updated via SVG `fill` attribute |
| Node stroke | None by default; `#3b82f6` 2px stroke on selected node |
| Node transition | `fill` changes are instant (no transition — 200ms update rate makes transition feel laggy) |
| Edge element | `<line>` per INFLUENCED_BY relationship |
| Edge stroke | `#6b7280` at opacity 0.4, stroke-width 1px |
| Edge entrance | CSS transition: `opacity 0 -> 1` over 600ms ease-out (D-14) |
| Background | `#0f1117` via CSS on container div |

Node sizing by bracket tier (D-10):

| Bracket Index | Archetype | Radius |
|---------------|-----------|--------|
| 0 | Quants | 5px |
| 1 | Degens | 6px |
| 2 | Sovereigns | 7px |
| 3 | Macro | 8px |
| 4 | Suits | 9px |
| 5 | Insiders | 10px |
| 6 | Agents | 11px |
| 7 | Doom-Posters | 12px |
| 8 | Policy Wonks | 13px |
| 9 | Whales | 14px |

Bracket clustering (D-07, D-08):
- 10 centroids arranged in a circle at radius = 35% of the smaller viewport dimension
- `d3.forceX` and `d3.forceY` pull nodes toward their bracket centroid with strength 0.3
- Centroids computed once at initialization from bracket list, never recomputed

Force simulation parameters:
- `d3.forceLink`: distance 60, strength 0.1
- `d3.forceManyBody`: strength -30
- `d3.forceCenter`: center of SVG viewport
- `d3.forceCollide`: radius = node radius + 2px padding
- Reheat (`simulation.alpha(0.3).restart()`) only on topology changes (new nodes, new edges) per D-06
- State updates (signal color, rationale) applied as direct DOM attribute mutations without touching the simulation

### AgentSidebar.vue

Fixed right panel for agent detail inspection (D-16, D-17, D-18).

| Property | Value |
|----------|-------|
| Width | 280px fixed |
| Background | `#1a1d27` |
| Position | Fixed right, full viewport height |
| Entrance | CSS transform `translateX(100%) -> translateX(0)` over 300ms ease-out |
| Exit | CSS transform `translateX(0) -> translateX(100%)` over 200ms ease-in |
| Border left | 1px solid `#374151` |
| Close button | Plain text "X" at 16px, top-right corner, 8px padding, hover color `#3b82f6` |
| Z-index | 10 (above graph SVG) |

Sidebar layout (top to bottom, `md` = 16px section gaps):

1. **Header row** — Agent name (Heading 18px/600, `#e5e7eb`), close button right-aligned
2. **Bracket label** — Label 12px/400, `#9ca3af`, e.g. "Quants"
3. **Signal chip** — Inline badge: Label 12px/400, signal color text on 15% opacity signal color background, rounded 4px, padding 4px 8px. Text is uppercase signal name ("BUY", "SELL", "HOLD")
4. **Divider** — 1px solid `#374151`, full width, 16px vertical margin
5. **Rationale section** — Label "Current Rationale" (Label 12px/400, `#9ca3af`), then rationale body text (Body 14px/400, `#e5e7eb`, line-height 1.5). Scrollable overflow if text exceeds available height.

Behavior:
- Only one sidebar open at a time (D-18)
- Clicking a different node switches sidebar content immediately (no close/reopen animation)
- Clicking empty graph space or close button triggers exit animation
- Graph container width shrinks by 280px when sidebar is open (D-16) — CSS `calc(100vw - 280px)` transition on graph container width over 300ms, matching sidebar entrance timing
- Content updates in real time as WebSocket snapshots arrive for the selected agent (D-17)

## Interaction States

### Graph — Empty (No Simulation Running)

The graph area shows the empty state centered in the viewport.

| Element | Specification |
|---------|---------------|
| Heading | "Waiting for Simulation" — Heading 18px/600, `#e5e7eb` |
| Body | "Start a simulation to see 100 agents deliberate in real time." — Body 14px/400, `#6b7280` |
| Alignment | Centered horizontally and vertically in the graph container |
| Pulse | A subtle opacity pulse animation (0.5 -> 1.0 -> 0.5) on the heading, 2s duration, infinite, `#3b82f6` color. Signals the system is alive and waiting. |

### Graph — Simulation Active

Nodes appear as they populate. Signal colors update at 200ms WebSocket cadence.

| Behavior | Specification |
|----------|---------------|
| Node appearance | Nodes render at initialization with PENDING fill (`#374151`), transition to signal color on first state update |
| Signal update | SVG `fill` attribute set directly from WebSocket payload `agent_states[id].signal` mapping |
| Round transition | When `round_num` changes in WebSocket payload, fetch `GET /api/edges/{cycle_id}?round={round_num}`, add new `<line>` elements with opacity 0, CSS transition to opacity 0.4 over 600ms |
| Edge persistence | Edges from prior rounds persist until cycle resets (simulation completes or new cycle starts) |

### Graph — Simulation Complete

Graph freezes in final state. Nodes retain last signal colors. No layout changes.

### Sidebar — Empty (No Agent Selected)

Sidebar is hidden (off-screen right). No placeholder visible in the graph area.

### Sidebar — Agent Selected

Sidebar slides in from right. Graph container shrinks. Content populated from WebSocket snapshot data for selected agent ID.

### Sidebar — No Rationale Yet

When agent is selected but has no rationale for the current round:

| Element | Specification |
|---------|---------------|
| Rationale area | "Click any agent node to inspect their reasoning." replaced by "Awaiting rationale for this round." — Body 14px/400, `#6b7280`, italic |

### Error State — WebSocket Disconnected

| Element | Specification |
|---------|---------------|
| Position | Fixed bottom-center of viewport, 32px from bottom edge |
| Background | `#1a1d27` with 1px solid `#ef4444` border |
| Padding | 8px 16px |
| Border radius | 4px |
| Text | "Connection Lost -- Attempting to reconnect to the simulation server." — Label 12px/400, `#ef4444` |
| Behavior | Appears after 3 consecutive failed WebSocket reconnect attempts. Disappears on successful reconnection. |
| Reconnect strategy | Exponential backoff: 1s, 2s, 4s, 8s, max 8s. Indefinite retry. |

## WebSocket Payload Reference

The frontend consumes JSON snapshots at 5Hz from `ws://localhost:8000/ws/state`. The schema is defined by `snapshot_to_json()` in `broadcaster.py`, which serializes `StateSnapshot` via `dataclasses.asdict()`.

Key fields the Vue app reads:

| JSON Path | Type | Frontend Usage |
|-----------|------|---------------|
| `phase` | string enum: `idle`, `seeding`, `round_1`, `round_2`, `round_3`, `complete`, `replay` | Show/hide empty state, determine simulation activity |
| `round_num` | int (0-3) | Trigger edge fetch on change |
| `agent_states` | `{agent_id: {signal: str, confidence: float}}` | Node fill color mapping |
| `agent_states[id].signal` | `"buy"` / `"sell"` / `"hold"` / `"parse_error"` / `null` | Map to signal colors; `null` and `parse_error` render as PENDING |
| `rationale_entries` | `[{agent_id, signal, rationale, round_num}]` | Update sidebar rationale if selected agent matches |
| `bracket_summaries` | `[{bracket, display_name, buy_count, sell_count, hold_count, ...}]` | Not used in Phase 31 (Phase 33 monitoring panels) |
| `governor_metrics` | object or null | Not used in Phase 31 (Phase 33 telemetry strip) |

## Edge Endpoint Reference

`GET /api/edges/{cycle_id}?round=N`

Response schema (D-13):
```json
{
  "edges": [
    {"source_id": "A_01", "target_id": "A_42", "weight": 0.85}
  ]
}
```

Frontend fetches this on `round_num` change in the WebSocket snapshot. The `cycle_id` is inferred from the simulation context (the frontend tracks the active cycle ID from initial WebSocket state or a separate mechanism — to be determined during planning).

## Animation Summary

| Animation | Property | Duration | Easing | Trigger |
|-----------|----------|----------|--------|---------|
| Edge fade-in | opacity 0 -> 0.4 | 600ms | ease-out | New round detected in WebSocket payload |
| Sidebar entrance | translateX(100%) -> translateX(0) | 300ms | ease-out | Node click |
| Sidebar exit | translateX(0) -> translateX(100%) | 200ms | ease-in | Close button or empty space click |
| Graph width shrink | width 100vw -> calc(100vw - 280px) | 300ms | ease-out | Sidebar opens |
| Graph width expand | width calc(100vw - 280px) -> 100vw | 200ms | ease-in | Sidebar closes |
| Empty state pulse | opacity 0.5 -> 1.0 -> 0.5 | 2000ms | ease-in-out | Continuous while idle, `#3b82f6` color on heading |

All animations use CSS transitions or CSS keyframes. No JavaScript-driven animation (D3 handles physics only, not visual transitions).
