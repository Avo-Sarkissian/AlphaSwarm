<objective>
Implement `RSSNewsProvider` — the real `NewsProvider` for Phase 38 — backed by `httpx.AsyncClient` for async HTTP fetch and `feedparser` for RSS parsing. Routes each entity to one of two sources: ticker symbols (regex `^[A-Z]{1,5}$`) fetch Yahoo Finance RSS (`https://finance.yahoo.com/rss/headline?s={ticker}`); everything else (geopolitical topics, sectors, etc.) fetches Google News RSS search (`https://news.google.com/rss/search?q={quote_plus(entity)}&hl=en-US&gl=US&ceid=US:en`). Every HTTP request sends `User-Agent: Mozilla/5.0 AlphaSwarm/6.0` (Pitfall 4 — Yahoo 429s without it). Feed text is passed to `feedparser.parse(response.text)` — never a URL directly (Pitfall 2). Entity filter is case-insensitive substring match (D-03). `max_age_hours` filter converts `entry.published_parsed` (a `time.struct_time`) to UTC datetime via `datetime.fromtimestamp(time.mktime(...), tz=UTC)` (Pitfall 3).

Threat T-38-01 (HIGH): URL injection via entity strings — `entity` comes from the seed rumor (untrusted input). Mitigation: (a) ticker path uses a strict regex `^[A-Z]{1,5}$` so only validated ticker chars reach the Yahoo URL; (b) Google News path wraps every entity in `urllib.parse.quote_plus(entity)` before string interpolation. Both defenses are asserted by dedicated unit tests.

Purpose: Close INGEST-02. This is the provider Phase 40 will call to assemble `ContextPacket.news`. Unit tests use mocked httpx responses; integration tests (Plan 03) hit real network under `enable_socket`.
Output: One new source module (`rss_provider.py`), one updated `__init__.py`, one updated `pyproject.toml` (feedparser dep + importlinter source_modules), one new unit test module.

"""INGEST-02: Real RSSNewsProvider — httpx async fetch + feedparser parse.

D-01: RSS-only news provider — no API key, no rate limits.
D-02: Dual-source routing — ticker regex -> Yahoo Finance RSS; else Google News RSS.
D-03: Case-insensitive substring match for entity filter.
D-04: feedparser + httpx.
D-09: staleness='fresh' on success (including empty entries); 'fetch_failed' on exception.
D-10: no caching.
D-19: NEVER raises — every failure mode returns _fetch_failed_news_slice(entity, 'rss').

Pitfall 2 (HIGH): feedparser.parse(url) uses a sync URL fetcher that trips
bozo=True with URLError in this environment. ALWAYS fetch bytes with
httpx.AsyncClient first, then pass response.text to feedparser.parse().

Pitfall 3: entry.published_parsed is time.struct_time (NOT datetime).
Convert with datetime.fromtimestamp(time.mktime(published_parsed), tz=UTC)
before comparing against max_age_hours.

Pitfall 4: Yahoo Finance RSS returns HTTP 429 without an explicit User-Agent.
Always set User-Agent: Mozilla/5.0 AlphaSwarm/6.0 on every fetch.

T-38-01 (HIGH — URL injection): entity strings come from the seed rumor
(untrusted input). Mitigations:
  - Ticker path: regex ^[A-Z]{1,5}$ guarantees only ticker-safe chars reach
    the Yahoo URL; anything else routes through Google News where...
  - Google News path: urllib.parse.quote_plus(entity) percent-encodes every
    reserved character before string interpolation.
Both mitigations are asserted by unit tests.
"""

from __future__ import annotations

import asyncio
import re
import time
from datetime import UTC, datetime
from typing import Any
from urllib.parse import quote_plus

import feedparser
import httpx

from alphaswarm.ingestion.providers import _fetch_failed_news_slice
from alphaswarm.ingestion.types import NewsSlice

_SOURCE = "rss"
_USER_AGENT = "Mozilla/5.0 AlphaSwarm/6.0"
_TICKER_RE = re.compile(r"^[A-Z]{1,5}$")
_REQUEST_TIMEOUT_S = 10.0


def _route_url(entity: str) -> str:
    """Ticker -> Yahoo Finance RSS; everything else -> Google News RSS.

    T-38-01: Ticker regex is a whitelist; Google News path quotes the entity.
    """
    if _TICKER_RE.match(entity):
        return f"https://finance.yahoo.com/rss/headline?s={entity}"
    return (
        f"https://news.google.com/rss/search?q={quote_plus(entity)}"
        "&hl=en-US&gl=US&ceid=US:en"
    )


def _entry_age_hours(entry: dict[str, Any]) -> float | None:
    """Pitfall 3: entry.published_parsed is time.struct_time; convert before arithmetic.

    Returns None if the entry has no published_parsed — in that case the age
    filter is skipped (we treat undated entries as potentially fresh).
    """
    published_parsed = entry.get("published_parsed")
    if published_parsed is None:
        return None
    ts = time.mktime(published_parsed)
    published_dt = datetime.fromtimestamp(ts, tz=UTC)
    return (datetime.now(UTC) - published_dt).total_seconds() / 3600.0


async def _fetch_one(
    client: httpx.AsyncClient, entity: str, max_age_hours: int
) -> NewsSlice:
    """Never-raise: any exception -> _fetch_failed_news_slice(entity, 'rss').

    Covered failure modes (any of which trips the outer except):
      - httpx.RequestError subclasses (timeout, DNS, connection refused)
      - httpx.HTTPStatusError from r.raise_for_status() (429 without UA, 5xx)
      - feedparser bozo exceptions surfacing as attribute errors
      - ANY unexpected exception — D-19 contract
    """
    try:
        url = _route_url(entity)
        r = await client.get(
            url,
            headers={"User-Agent": _USER_AGENT},
            timeout=_REQUEST_TIMEOUT_S,
        )
        r.raise_for_status()
        feed = feedparser.parse(r.text)
        needle = entity.lower()
        headlines: list[str] = []
        for entry in feed.entries:
            title = entry.get("title", "")
            if not title or needle not in title.lower():
                continue
            age = _entry_age_hours(entry)
            if age is not None and age > max_age_hours:
                continue
            headlines.append(title)
        # Research Open Question 2: 0 entries from Yahoo with HTTP 200 is fresh,
        # not fetch_failed. We distinguish "nothing to report" from "could not fetch".
        return NewsSlice(
            entity=entity,
            headlines=tuple(headlines),

    A single httpx.AsyncClient is constructed per get_headlines call and
    shared across every entity fetch (via asyncio.gather) for connection

        self, entities: list[str], *, max_age_hours: int = 72

            return {}

class _MockResponse:

        self.text = text

        if self.status_code >= 400:

                f"status {self.status_code}",

class _MockAsyncClient:
    """Replacement for httpx.AsyncClient.

    Usage: _MockAsyncClient.configure(responder=callable | dict); callable
    receives the url and returns a _MockResponse OR raises. Dict maps url->response.
    Recorded calls are available in .calls.
    """

    responder: Any = None  # set via configure()
    calls: list[dict[str, Any]] = []

    @classmethod
    def configure(cls, responder: Any) -> None:
        cls.responder = responder
        cls.calls = []

    def __init__(self, *args: Any, **kwargs: Any) -> None:
        self._init_args = args
        self._init_kwargs = kwargs

    async def __aenter__(self) -> "_MockAsyncClient":
        return self

    async def __aexit__(self, *_: Any) -> None:
        return None

    async def get(self, url: str, *, headers: dict[str, str], timeout: float) -> _MockResponse:
        type(self).calls.append({"url": url, "headers": dict(headers), "timeout": timeout})
        responder = type(self).responder
        if callable(responder):
            maybe = responder(url)
            if isinstance(maybe, Exception):
                raise maybe
            return maybe
        if isinstance(responder, dict):
            return responder[url]
        raise AssertionError("_MockAsyncClient.responder not configured")

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| seed rumor entity string -> URL construction | Entities are extracted from the seed rumor (untrusted user input). A malicious entity like `"EV&cmd=drop"` or a pathological Unicode string must not be able to craft a harmful URL or SSRF target. |
| external RSS endpoint -> feedparser -> NewsSlice | feedparser parses external XML. Malformed feeds, bozo flags, or XML bombs must not crash the provider or the aggregation loop. |
| event loop -> httpx async HTTP | All RSS fetches run through httpx.AsyncClient (async-first library). No sync urllib call on the event loop. |
| tests -> real network | Unit tests gated by pytest-socket --disable-socket. Every httpx.AsyncClient access is monkeypatched. |
