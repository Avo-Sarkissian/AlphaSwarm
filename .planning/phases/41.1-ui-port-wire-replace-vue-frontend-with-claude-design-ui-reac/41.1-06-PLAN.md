<objective>
Fix NR-1 (WebSocket connects to Vite dev server :5173 instead of FastAPI :8000) by replacing the `window.location.host` URL construction in `useWebSocket.ts` with an absolute origin sourced from `import.meta.env.VITE_BACKEND_WS`. Default the env to `ws://localhost:8000` for local dev. This unblocks NR-4 (rationale feed empty) which is downstream of the same broken connection.

Purpose: Session 3 UAT confirmed the WS hook tries to upgrade `ws://localhost:5173/ws/state`. Vite's dev proxy (`vite.config.ts` already proxies `/ws` to `localhost:8000`) was supposed to mediate this, but the live behavior over a 54-minute run shows the upgrade never reached FastAPI — every panel that depends on the live frame (force graph, brackets, rationale feed, KPI deltas) ran empty until REST fallback at FINAL. Eliminating the proxy round-trip and hardcoding an absolute origin is the surgical fix.

Output: A WS hook that, regardless of which port `npm run dev` serves the SPA on, always upgrades to FastAPI directly. NR-1 closed; NR-4 unblocked (Plan 09 verifies the rationale feed actually populates).
</objective>
