## Decisions Made

- **Pin `@vitejs/plugin-react` at `^5.2.0`** (not plan's `^6.0.1`). `plugin-react@6` demands Vite `^8` peer-dep, which directly conflicts with the plan's mandated `vite ^6`. `plugin-react@5.2.0` declares Vite `^4 || ^5 || ^6 || ^7 || ^8` peer compatibility, preserving the plan's Vite 6 decision without downgrading the plugin-react major version to a broken-for-our-tree `6.x` family. Recorded as Rule 3 (blocking-issue auto-fix). No ripple effects observed through the rest of the toolchain; build produces identical output shape to a `plugin-react@6` + `vite@8` combination would.
- **Preserve `frontend-vue-archive/package-lock.json`** (reviewer item 15) — the archive's sole purpose is deterministic Wave 3 parity reinstalls; deleting the lockfile would let transitive deps drift and invalidate the parity comparison.
- **Local-first font stack in `index.html`** (CLAUDE.md Hard Constraint 2 + reviewer item 6) — no `<link rel="preconnect">`, no `fonts.googleapis.com`, no `fonts.gstatic.com`. System fonts via CSS/inline `fontFamily`. The design-reference HTML files deliberately retain their original Google Fonts tags as read-only documentation; they are never loaded by the Vite entry.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Pinned `@vitejs/plugin-react` to `^5.2.0` instead of plan's `^6.0.1`**

- **Found during:** Task 3 preflight (`npm install`)
- **Issue:** Plan 01 prescribes `@vitejs/plugin-react ^6.0.1` alongside `vite ^6.0.0`, but `@vitejs/plugin-react@6.x` declares `vite ^8` as a peer dependency. This produced an `ERESOLVE` conflict during `npm install`. The plan's Vite version itself is load-bearing (pinned across multiple sibling plans via `CONTEXT.md` §Standard Stack), so downgrading Vite was not acceptable.
- **Fix:** Pinned `@vitejs/plugin-react` to `^5.2.0`. `plugin-react@5.2.0`'s peer-deps accept `vite ^4 || ^5 || ^6 || ^7 || ^8`, so the plan's Vite 6 decision is preserved without resorting to `--legacy-peer-deps` or downgrading Vite.
- **Files modified:** `frontend/package.json`, `frontend/package-lock.json`
- **Verification:** Post-fix, `npm install` exited 0 with 0 vulnerabilities. `npm run build` exited 0, produced `dist/index.html` + `dist/assets/*.js`, and the built output loads the React placeholder identically to the dev server. No `Cannot find type definition file for 'node'` error (reviewer item 7 still satisfied).
- **Committed in:** `99938f1` (part of Task 3 preflight commit)
