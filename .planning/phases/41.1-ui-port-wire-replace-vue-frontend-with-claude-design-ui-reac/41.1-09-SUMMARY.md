## NR-6 Architecture

### New Hook (`frontend/src/hooks/useAdvisoryAutoTrigger.ts`)

```ts
const advisoryAutoTriggered = new Set<string>();      // module-scope registry
export function hasAdvisoryBeenTriggered(cycleId): boolean
export function markAdvisoryTriggered(cycleId): void
export function useAdvisoryAutoTrigger(): void
```

Behavior:

1. Reads `lastFrame` from `useConnection()` and `cycleId` from `useCurrentCycle()`.
2. `useRef<lastSeenPhase>` tracks the previous frame's phase.
3. On each effect tick: only fire when `(was non-complete) && (is complete) && !registry.has(cycleId)`.
4. `add(cycleId)` BEFORE the POST so a synchronous re-render cannot double-dispatch.
5. `catch (ApiError 409)` → keep the mark (synthesis already running; polling picks up the result).
6. `catch (else)` → `delete(cycleId)` so the user's manual modal click can retry.

### Wire-up (`frontend/src/App.tsx`)

```tsx
import { useAdvisoryAutoTrigger } from './hooks/useAdvisoryAutoTrigger';
// ...
<EdgesProvider round={round}>
  <AdvisoryAutoTriggerMount />   {/* invisible — returns null */}
  <DashboardShell />
</EdgesProvider>
// ...
function AdvisoryAutoTriggerMount() {
  useAdvisoryAutoTrigger();
  return null;
}
```

The component returns `null`. It exists solely to mount the hook **inside** the provider tree, so `useConnection()` and `useCurrentCycle()` resolve correctly. The mount intentionally lives only in the main return path (after the `if (!lastFrame)` early return); auto-trigger is meaningless without a frame.

### Modal short-circuit (`frontend/src/components/modals.jsx`)

```jsx
useEffect(() => {
  if (!cycleId || kicked) return;
  setKicked(true);
  // NR-6: if useAdvisoryAutoTrigger already dispatched for this cycleId
  // (auto-fired on phase=complete), this manual open is a no-op.
  if (hasAdvisoryBeenTriggered(cycleId)) return;
  markAdvisoryTriggered(cycleId);
  advisoryGenerate(cycleId).catch((e) => {
    if (e instanceof ApiError && e.status === 409) return;
    setKickError(e);
  });
}, [cycleId, kicked]);
```

Two-layer guard:
- `kicked` (React state) — in-component de-dup; blocks repeat fires within the same modal mount.
- `hasAdvisoryBeenTriggered(cycleId)` (module Set) — cross-mount de-dup; blocks the modal-open-after-auto-trigger path.

When the user opens the modal BEFORE the cycle reaches FINAL, `kicked` is false AND the registry is empty → the modal kicks (and marks). The auto-trigger then sees the registry mark on the FINAL transition and skips. Polling resolves the result for both paths.
