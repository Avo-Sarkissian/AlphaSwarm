**Six modals, cycle history, and lifecycle overlays now read from the live FastAPI backend via key-polled REST wrappers with ApiError 404→pending branching and DOMPurify-sanitized markdown rendering.**

## Accomplishments

- **InterviewModal** — POSTs `{message}` to `/api/interview/{agent_id}` via `askAgent()`; renders full response as one block (KR-41.1-11 non-streaming accepted). Handles 409 (phase not complete) + 503 (services down) with `formatApiError` user-facing strings.
- **ReportModal** — Kicks `POST /api/report/{cycle_id}/generate` once per cycleId, then polls `GET /api/report/{cycle_id}` with `usePolling({ key: \`report:${cycleId}\`, ... })`; 404 stays in "generating" state, 500 surfaces error; content rendered via `DOMPurify.sanitize(marked.parse(content))`.
- **AdvisoryModal** — Same pattern (`advisory:{cycleId}` key). Holdings fetched via `fetchHoldings()` from `/api/holdings`, which tolerates 404/503 with an empty snapshot. Markdown body via DOMPurify+marked.
- **ShockDrawer** — `onInject` calls Plan 02's `simShock(text)` (canonical `/api/simulate/shock`). Shows busy state + inline error on failure; does not close on error so user can retry.
- **ReplayBar** — `replayAdvance()` + `replayStop()`; round read from `useTelemetry().roundNum`. **Rewind disabled with `KR-41.1-06` comment** (no backend rewind endpoint).
- **CyclePickerModal** — `listCycles()` replaces `window.AS_DATA.REPLAY_CYCLES`. Loading + empty + error states.
- **CycleHistory** — Fetches via `listCycles()`; em-dash placeholders for consensus/flips/duration/shocks columns under `KR-41.1-07` (list endpoint omits those fields). Starred filter removed (no starred field on `CycleItem`).
- **CompareModal** — Accepts `CycleItem` objects; round-tally bar chart kept as illustrative fixture (KR-41.1-07) until backend exposes per-round consensus.
- **SeedingState** — Reads `useAgents().agents` directly; deleted `setInterval` fake counter and `BRACKETS` fixture.
- **MemoryPausedState** — `memPct` prop sourced from telemetry via `LifecycleOverlay`.
- **ErrorState** — Reads `useConnection().reconnectFailed`; KR-41.1-08 comment marking illustrative log rows; `console.error(error)` on surfaced errors.
- **LifecycleOverlay** — Single entry wrapping the decision table (connection.reconnectFailed > phase=='idle' > phase=='seeding' > mem≥threshold > null). No new hook (per prompt `<do_not>`).
- **TweaksPanelLoader.tsx** — `React.lazy` + `import.meta.env.DEV` branch; belt-and-braces `if (!import.meta.env.DEV) return null`. Production bundle excludes the dev branch.
- **tweaks.jsx** — Consumes split contexts (useTelemetry + useAgents); holdings preview dynamic-imports `../mocks/holdings` under DEV; "Force MemoryPaused (dev)" button dispatches `as:force-mem-pause` for Plan 05 UAT.
- **settings.jsx** — Added `KR-41.1-11` header comment marking persistence deferred; local state only.

## Decisions Made

- Did not create `hooks/useLifecycleOverlay.ts` because the prompt `<do_not>` block forbade new hooks. Inlined the decision table at the bottom of `states.jsx` inside `LifecycleOverlay` directly. Functionality is identical; behavior parity kept.
- Passed the raw `HoldingsSnapshot` through `fetchHoldings()` instead of coercing the backend's decimal-string `qty` into a number, to preserve fractional-share precision (matches backend's Pitfall 1).
- Kept the illustrative `roundsA/roundsB` bar-chart fixtures in CompareModal as a design parity placeholder; marked KR-41.1-07 inline. Cycle-ID pairing UX works with live data.
- `CompareModal` now takes `CycleItem` objects directly (instead of IDs) so the parent's selected-cycles map yields both without a second lookup.
- `formatApiError` helper centralizes the "extract `detail.message` from ApiError body" pattern — modals stay DRY and consistent.
- Adapted to the actual `usePolling` return shape (`{data, error, attempt, loading}`) instead of the plan-doc's theoretical `{value, status, error}`. Plan 02 shipped the simpler shape; the plan-doc described a future signature that was never implemented. Both semantic modes (loading / ready / error) are still achievable from `data`/`error`.
- Chose NOT to edit `app_v2.jsx` / `app.jsx` to swap `TweaksPanel` direct import for `TweaksPanelLoader` — those files are Plan 03's scope per the prompt `<do_not>` block.
