# Phase 41.1 Plan 06: NR-1 WebSocket Origin Fix Summary

WebSocket hook now connects to FastAPI directly via `import.meta.env.VITE_BACKEND_WS` instead of `window.location.host` — closes NR-1 (WS connecting to Vite dev server :5173) and unblocks NR-4 (rationale feed empty downstream).

## Old vs New URL Construction (3-line diff)

**Before (NR-1 broken — `frontend/src/hooks/useWebSocket.ts` lines 34-35):**
```ts
const proto = window.location.protocol === 'https:' ? 'wss:' : 'ws:';
const url = `${proto}//${window.location.host}${path}`;
// → ws://localhost:5173/ws/state when SPA served by Vite dev server (BROKEN)
```

**After (`frontend/src/hooks/useWebSocket.ts` lines 34-38):**
```ts
// NR-1 FIX: connect to FastAPI directly (absolute origin), NOT to whatever
// host happens to be serving the SPA (e.g. Vite dev server on :5173).
// Default 'ws://localhost:8000' keeps local-first dev working without env setup.
const base = (import.meta.env.VITE_BACKEND_WS as string | undefined) || 'ws://localhost:8000';
const url = `${base}${path}`;
// → ws://localhost:8000/ws/state regardless of SPA host port
```

## New Env Var Contract

| Var | Type | Default | Purpose |
|-----|------|---------|---------|
| `VITE_BACKEND_WS` | string | `ws://localhost:8000` | Absolute origin (incl. protocol) the WS hook concatenates with `/ws/state`. Honors `wss://` for HTTPS deploys. |

- Set in `frontend/.env.development` (loaded only in `mode === 'development'`).
- Typed in `frontend/src/env.d.ts` via extension of `ImportMetaEnv`.
- Must remain `VITE_*`-prefixed to be exposed to client bundle by Vite.
- Re-usable: Plan 11 connection-state banner can read the same env to label the connection.

## Vite Proxy Status

`frontend/vite.config.ts` `server.proxy['/ws']` entry is intentionally **left in place**:

```ts
'/ws': { target: 'ws://localhost:8000', ws: true },
```

Rationale: the proxy is now inert for `useWebSocket.ts` (the hook bypasses Vite entirely with an absolute origin), but a future plan that reverts to host-relative URLs would silently break without it. Keeping the entry preserves revert safety with zero behavior change for this plan.
