# Phase 26 — UI Design Contract

> Visual and interaction contract for the ShockInputScreen TUI overlay. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope

This phase delivers a single new Textual overlay screen — **ShockInputScreen** — pushed between simulation rounds when the StateStore shock-window event fires. Copy/paste target: `RumorInputScreen` at `src/alphaswarm/tui.py:385-450`. The contract below locks every visual and interaction token reused from the existing dark TUI theme so the shock overlay is visually indistinguishable from the rest of AlphaSwarm (minimalist, one-purpose-at-a-time).

Out of scope for the UI contract:
- Governor suspend/resume wiring (non-visual; D-01..D-03)
- StateStore queue/event primitives (non-visual; D-04..D-07)
- Neo4j `ShockEvent` persistence (non-visual; D-11..D-14)
- Before/after consensus comparison (Phase 27)

## Spacing Scale

Textual uses cell units (1 cell ≈ 1 character width / 1 line height), not pixels. The declared scale below is expressed in **cells** and maps 1:1 to Textual's `padding`, `margin`, and `width`/`height` integer values. All declared values are multiples of 1 (the base unit), which maps conceptually to the 4px scale at a ~4px-per-cell mental model.

| Token | Value (cells) | px-equivalent | Usage |
|-------|---------------|---------------|-------|
| xs | 1 | 4px | Inline padding inside the input border (`padding: 0 1`) |
| sm | 2 | 8px | Vertical gap above the hint line (`margin-top: 1`), subtitle bottom margin (`margin-bottom: 2`) |
| md | 3 | 16px | Overlay container horizontal padding (`padding: 2 3`) |
| lg | 6 | 24px | Not used on this screen (reserved) |
| xl | 8 | 32px | Not used on this screen (reserved) |

**Fixed overlay dimensions (locked to match `RumorInputScreen`):**
- Container width: **72 cells** (`width: 72`)
- Container height: **auto** (content-driven)
- Container padding: **2 cells vertical, 3 cells horizontal** (`padding: 2 3`)
- Title margin-bottom: **1 cell**
- Subtitle margin-bottom: **2 cells**
- Hint margin-top: **1 cell**

**Exceptions:** none. Every spacing value reuses existing `RumorInputScreen` tokens to guarantee visual parity.

## Typography

Textual has no px-based type scale; all text is terminal-monospace with weight/style markers only. The declared "roles" below map to Rich/Textual `text-style` values.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display (title) | 1× (terminal default) | `bold` | 1 cell | "Inject Breaking Event" header |
| Body (subtitle) | 1× (terminal default) | regular | 1 cell | "Shock the swarm before Round N" context line |
| Input | 1× (terminal default) | regular | 1 cell | Input field text and placeholder |
| Hint | 1× (terminal default) | regular | 1 cell | "Enter to inject · Esc to skip" footer |

**Weights declared:** exactly two — `bold` (display only) and regular (body, input, hint). No other `text-style` values (no `italic`, no `underline`).

**Why two sizes only:** Terminal fonts are monospace fixed-size. The "scale" is achieved through color contrast and weight, not font size. This is the standard Textual aesthetic the project has used since Phase 9.

| Accent (10%) | `#4FC3F7` (AlphaSwarm cyan) | Container border, title text |

**Accent reserved for (explicit list):**
1. Overlay container border (1-cell solid `#4FC3F7`)
2. Title text `"Inject Breaking Event"` (bold `#4FC3F7`)

**Why no destructive color on the screen itself:** Injecting a shock is a **non-destructive** authoring action — it adds data, does not delete or overwrite. ## Interaction Contract

| Interaction | Trigger | Effect |
|-------------|---------|--------|
| Open overlay | `_poll_snapshot()` detects `state_store.shock_window.is_set()` is True (D-16) | `self.push_screen(ShockInputScreen(next_round=N))` |
| Auto-focus input | `on_mount()` | `self.query_one("#shock-input", Input).focus()` |
| Submit shock | `Input.Submitted` with non-empty trimmed value | `self.dismiss(shock_text)` → caller puts text onto `state_store.shock_queue` |
| Skip shock (empty submit) | `Input.Submitted` with empty trimmed value | `self.dismiss(None)` — treated as "no shock for this round" (D-07) |
| Dismiss without submit | `Escape` key binding | `self.dismiss(None)` — same as empty submit |
| Resume simulation | Queue receives `None` OR non-empty text | Governor `resume()`, shock-window event cleared, next round starts |

**Key bindings (exactly two, mirroring `RumorInputScreen`):**
```python
BINDINGS = [
    ("escape", "skip_shock", "Skip"),
    # Enter submit is handled by Input.Submitted, not a Binding
]
```

**Focus order:** Input is the only focusable widget on the screen. No tab navigation needed.

**Blocking semantics:** While ShockInputScreen is pushed, the dashboard behind it is frozen (governor is suspended — D-05). The overlay is modal by virtue of being a Textual `Screen`; the dashboard cannot receive input until dismissal.

## State Contract

ShockInputScreen has **one** visual state path (input accepting text) and two termination paths.

| State | Visual | Trigger → Next |
|-------|--------|----------------|
| Ready (default) | Empty Input with placeholder visible; auto-focused | Trigger: mount. Next: User types → Typing |
| Typing | Input shows user text; placeholder hidden | Trigger: keystroke. Next: Enter → Submitted / Esc → Skipped |
| Submitted | (Screen dismisses immediately — no visible transition state) | Trigger: non-empty Enter. Next: dashboard resumes with shock applied |
| Skipped | (Screen dismisses immediately — no visible transition state) | Trigger: Esc OR empty Enter. Next: dashboard resumes with no shock |

**No loading state.** Submission is a local `dismiss()` call — the downstream Neo4j write and governor resume happen **after** the screen is already gone. The user never waits on the modal.

**No error state on the screen.** Neo4j write failures are surfaced via `self.notify(..., severity="error")` on the dashboard after the screen dismisses (consistent with how Phase 14 Interview and Phase 15 Report handle post-dismiss errors).

**No empty state.** The screen itself is the empty state — it exists specifically to solicit input.

**Voice rules (applied):**
1. **Imperative verb + concrete noun** for CTA: "inject" + "Breaking Event" (not "Submit Shock" / not "OK")

3. **Never "Click":** the TUI has no mouse affordance for this screen; copy references keyboard only (`Enter`, `Esc`)
4. **Consistency with RumorInputScreen:** "Enter to start · Esc to quit" → "Enter to inject · Esc to skip" (same structure, domain-appropriate verbs)

## Accessibility

**Screen reader / terminal emulator support:**
- Textual uses native terminal rendering; Textual 8.x exposes screen content through the terminal's text buffer, so any screen reader that reads terminals (e.g. VoiceOver reading the Terminal.app buffer) will read title, subtitle, placeholder, and hint in source order.

**Keyboard-only operation:**
- Auto-focus on mount means no Tab is required to start typing
- Enter submits (standard terminal semantic)
- Esc dismisses (standard terminal modal semantic; consistent with `RumorInputScreen` and `InterviewScreen`)
- **Zero mouse dependency** — there is no button to click

**Contrast (WCAG AA checked against terminal rendering):**
- `#E0E0E0` on `#1E1E1E` (input body text on card): contrast ratio 12.3:1 — **PASS AAA**
- `#4FC3F7` on `#1E1E1E` (title and border on card): contrast ratio 8.9:1 — **PASS AAA**
- `#78909C` on `#1E1E1E` (subtitle and hint on card): contrast ratio 4.6:1 — **PASS AA normal text**

**Focus indicator:** Textual's default Input focus indicator (cursor) is sufficient; no custom focus ring required.

## Acceptance Criteria (for gsd-ui-checker)

A correct implementation of ShockInputScreen will:

1. Render a centered 72-cell-wide card with 1-cell `#4FC3F7` border on a `#121212` backdrop
2. Show exactly four widgets: bold cyan title, muted subtitle, Input with placeholder, muted hint
3. Use only colors from the declared palette — no new hex codes introduced
4. Auto-focus the Input on mount (zero keystrokes before typing)
5. Dismiss with the trimmed text on Enter (submitted path)
6. Dismiss with `None` on Esc **and** on empty Enter (skipped path — D-07)
7. Display the correct round number in subtitle via constructor arg
8. Be invoked from `_poll_snapshot()` when `state_store.shock_window.is_set()` transitions to True
9. Not introduce any new CSS selector or text-style not listed in this contract
10. Not render any mouse-clickable button, toolbar, or secondary action — keyboard-only
