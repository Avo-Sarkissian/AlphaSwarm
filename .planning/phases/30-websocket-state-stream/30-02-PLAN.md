Current src/alphaswarm/web/app.py lifespan() structure:
```python
@asynccontextmanager
async def lifespan(app: FastAPI) -> AsyncGenerator[None, None]:
    settings = AppSettings()
    brackets = load_bracket_configs()
    personas = generate_personas(brackets)
    app_state = create_app_state(settings, personas, with_ollama=True, with_neo4j=True)
    sim_manager = SimulationManager(app_state)
    connection_manager = ConnectionManager()

    app.state.app_state = app_state
    app.state.sim_manager = sim_manager
    app.state.connection_manager = connection_manager

    log.info("lifespan_started", phase="idle")

    yield

    # Teardown
    if app_state.graph_manager is not None:
        await app_state.graph_manager.close()

    log.info("lifespan_stopped")
```

Required modifications:
1. Add `import asyncio` to stdlib imports (check if already present — do not duplicate)
2. Add import: `from alphaswarm.web.broadcaster import start_broadcaster`
3. Add import: `from alphaswarm.web.routes.websocket import router as ws_router`
4. After `app.state.connection_manager = connection_manager` in lifespan(), add:
   ```python
   broadcaster_task = start_broadcaster(app_state.state_store, connection_manager)
   app.state.broadcaster_task = broadcaster_task
   ```
   CRITICAL: start_broadcaster receives the same `connection_manager` local variable that was
   just stored on app.state.connection_manager. This is how object identity is guaranteed:
   both ws_state (reads websocket.app.state.connection_manager) and the broadcaster
   (holds a reference to `connection_manager`) operate on the exact same object.
5. In teardown (after `yield`, BEFORE graph_manager.close()), add:
   ```python
   broadcaster_task.cancel()
   try:
       await broadcaster_task
   except asyncio.CancelledError:
       pass
   ```
   Ordering matters: cancel broadcaster BEFORE closing graph_manager, because the broadcaster
   loop calls state_store.snapshot() which may reference state that graph_manager teardown
   would invalidate.

<threat_model>
This plan modifies only app.py — the wiring layer. No new security surface.

| Threat | Risk | Mitigation |
|--------|------|------------|
| Broadcaster and ws endpoint use different connection_manager | Medium | Object identity guaranteed by passing same local variable to start_broadcaster and storing it on app.state; test_ws_state_same_connection_manager (Plan 01) verifies this; lifespan code comment documents the invariant |
| Broadcaster task holds open after shutdown | Medium | Cancel in lifespan teardown BEFORE graph_manager.close() — ordering prevents use-after-close on state_store |
| ws_router prefix collision with /api routes | Low | ws_router registered without prefix; /ws/state does not collide with /api/* routes |
| Multiple broadcaster tasks created | Low | lifespan is a singleton context — asynccontextmanager guarantees single execution per app instance |
| Production wiring breaks silently while test-app tests pass | Medium | test_create_app_ws_route_registered exercises real create_app() path and confirms /ws/state is registered |
| Unauthenticated WebSocket on non-loopback interface | Low | Per D-10: local dev only. alphaswarm web must bind to 127.0.0.1 (not 0.0.0.0) so the endpoint is not reachable from the network. The CLI launch command should use --host 127.0.0.1 or ensure AppSettings default is 127.0.0.1. |

ASVS L1: No auth on /ws/state per D-10 (explicit user decision, local dev only). Binding to 127.0.0.1 enforces the local-only contract at the network layer.
</threat_model>
