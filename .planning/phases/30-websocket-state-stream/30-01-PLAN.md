<objective>
Create the broadcaster module and WebSocket route that form the core of the 5Hz state stream, with test isolation that prevents broadcaster timing from making tests flaky or slow.

Purpose: These two new files are the Phase 30 implementation surface. The broadcaster serializes StateSnapshot to JSON every 200ms — critically, it calls drain_rationales() and OVERRIDES the snapshot dict's rationale_entries so the wire format matches the UI-SPEC contract. The WebSocket route registers clients with ConnectionManager and detects disconnects. Both files consume Phase 29 infrastructure without modifying it.

Tests use a direct-inject approach for WebSocket tests: the broadcaster does NOT start inside _unit_lifespan (which serves unrelated health/simulation tests). A separate _ws_lifespan starts the broadcaster only for tests that exercise /ws/state. The receive-snapshot test uses a seeded broadcast (calling connection_manager.broadcast() directly before receive_text()) to avoid any dependency on real 200ms ticker timing.

Output: src/alphaswarm/web/broadcaster.py, src/alphaswarm/web/routes/websocket.py, and updated tests/test_web.py with 5 new tests + _ws_lifespan helper.

<threat_model>
Phase 30 is a local-dev-only WebSocket stream (D-10: unauthenticated). Threat surface is minimal:

| Threat | Risk | Mitigation |
|--------|------|------------|
| Malicious client floods receive loop | Low (local dev only) | receive_text() naturally rate-limited by TCP; no application-layer mitigation needed |
| Client holds connection indefinitely | Low (local dev) | ConnectionManager bounded queue (100 msgs) + drop-oldest prevents memory growth |
| Serialization exception leaks stack trace | Low | `except Exception` in broadcast loop catches and logs with throttle counter; next tick proceeds — no trace exposed to client |
| CancelledError swallowed → zombie task | Medium | Use `except Exception` (not BaseException) in broadcast loop; CancelledError propagates to asyncio.sleep(0.2) |
| Rationale entries silently dropped | Medium | snapshot_to_json explicitly overrides d["rationale_entries"] after asdict(snap); tested by test_snapshot_to_json |
| Broadcaster and ws endpoint use different connection_manager | Medium | test_ws_state_same_connection_manager verifies object identity; Plan 02 explicitly documents the wiring |
| Noisy log spam on repeated failures | Low | Error counter throttles logging to once per 10 failures |

ASVS L1 controls applied: no sensitive data in error logs (snapshots contain only simulation metrics), no auth tokens (local dev), no PII in wire format.
</threat_model>

    ```python
        """FastAPI test app for WebSocket tests. Starts broadcaster (unlike _unit_lifespan)."""
        from contextlib import asynccontextmanager
        from collections.abc import AsyncGenerator
        import asyncio

        from alphaswarm.app import create_app_state
        from alphaswarm.config import AppSettings, generate_personas, load_bracket_configs
        from alphaswarm.web.broadcaster import start_broadcaster
        from alphaswarm.web.connection_manager import ConnectionManager
        from alphaswarm.web.routes.websocket import router as ws_router
        from alphaswarm.web.simulation_manager import SimulationManager

        @asynccontextmanager
        async def _ws_lifespan(app: FastAPI) -> AsyncGenerator[None, None]:
            settings = AppSettings(_env_file=None)  # type: ignore[call-arg]
            brackets = load_bracket_configs()
            personas = generate_personas(brackets)
            app_state = create_app_state(settings, personas, with_ollama=False, with_neo4j=False)
            sim_manager = SimulationManager(app_state)
            connection_manager = ConnectionManager()

            app.state.app_state = app_state
            app.state.sim_manager = sim_manager
            app.state.connection_manager = connection_manager

            # Start broadcaster — needed for WebSocket tests
            broadcaster_task = start_broadcaster(app_state.state_store, connection_manager)
            app.state.broadcaster_task = broadcaster_task

            yield

            broadcaster_task.cancel()
            try:
                await broadcaster_task
            except asyncio.CancelledError:
                pass

            if app_state.graph_manager is not None:
                await app_state.graph_manager.close()

        app = FastAPI(title="AlphaSwarm-WS-Test", lifespan=_ws_lifespan)
        app.include_router(ws_router)  # No prefix — /ws/state is the full path
        return app
    ```

    _broadcast_loop with log throttling (per review concern #6):
    ```python
    async def _broadcast_loop(
        state_store: StateStore,
        connection_manager: ConnectionManager,
    ) -> None:
        log = structlog.get_logger(component="web.broadcaster")
        consecutive_failures = 0
        while True:
            try:
                message = snapshot_to_json(state_store)
                connection_manager.broadcast(message)
                if consecutive_failures > 0:
                    log.info("broadcast_recovered", after_failures=consecutive_failures)
                    consecutive_failures = 0
            except Exception:
                consecutive_failures += 1
                # Log once per 10 consecutive failures to avoid 5Hz spam
                if consecutive_failures == 1 or consecutive_failures % 10 == 0:
                    log.exception("broadcast_tick_error", consecutive=consecutive_failures)
            await asyncio.sleep(0.2)
    ```

    CRITICAL: `except Exception` NOT `except BaseException` or `except asyncio.CancelledError`. The `CancelledError` is a BaseException in Python 3.8+; it must NOT be caught inside the loop body. It propagates naturally through `await asyncio.sleep(0.2)` to allow clean lifespan teardown.

    Full file structure:
    - `from __future__ import annotations` at top
    - stdlib imports: asyncio, dataclasses, json
    - third-party: structlog
    - local: from alphaswarm.state import StateStore; from alphaswarm.web.connection_manager import ConnectionManager
    - Exports: snapshot_to_json, start_broadcaster (private: _broadcast_loop)

    Note on timing: The loop runs as "work + asyncio.sleep(0.2)". True 5Hz means each tick fires approximately every 200ms plus the time to call snapshot()+broadcast() (microseconds for 7.3KB payload). This is "approximately 5Hz" per the roadmap goal — acceptable.

    ```python
    from __future__ import annotations

    import structlog
    from fastapi import APIRouter, WebSocket
    from starlette.websockets import WebSocketDisconnect

    log = structlog.get_logger(component="web.ws_state")
    router = APIRouter()

    @router.websocket("/ws/state")
    async def ws_state(websocket: WebSocket) -> None:
        """Accept WebSocket connection and enter receive loop for disconnect detection.

        Data flows server-to-client only: the broadcaster task calls
        connection_manager.broadcast() which enqueues messages to each client's
        writer task. This receive loop exists solely to detect client disconnects.

        Assumptions:
        - receive_text() is used (not receive()) — tolerated because this endpoint
          is local-dev-only (D-10) and clients are expected to be browser WebSocket
          connections or wscat, neither of which sends binary frames.
        """
        connection_manager = websocket.app.state.connection_manager
        log.info("ws_state_connected", client=str(websocket.client))
        await connection_manager.connect(websocket)
        try:
            while True:
                await websocket.receive_text()  # Blocks; exists solely for disconnect detection
        except WebSocketDisconnect:
            pass
        finally:
            log.info("ws_state_disconnected", client=str(websocket.client))
            await connection_manager.disconnect(websocket)
    ```

    Per D-10: no authentication, no origin checking — local dev only.

    Use `from __future__ import annotations` at top.

    Note on receive_text() vs receive(): receive_text() raises WebSocketDisconnect on disconnect and raises an error if client sends binary frames. Since this endpoint is local-dev-only (D-10) and serves browser clients and wscat (text-only), receive_text() is appropriate. If binary tolerance is ever needed, switch to websocket.receive() which handles both text and binary frames.
