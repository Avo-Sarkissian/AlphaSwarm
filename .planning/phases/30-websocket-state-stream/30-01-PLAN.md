<threat_model>
Phase 30 is a local-dev-only WebSocket stream (D-10: unauthenticated). Threat surface is minimal:

| Threat | Risk | Mitigation |
|--------|------|------------|
| Malicious client floods receive loop | Low (local dev only) | receive_text() naturally rate-limited by TCP; no application-layer mitigation needed |
| Client holds connection indefinitely | Low (local dev) | ConnectionManager bounded queue (100 msgs) + drop-oldest prevents memory growth |
| Serialization exception leaks stack trace | Low | `except Exception: log.exception(...)` in broadcast loop catches and logs; next tick proceeds normally — no trace exposed to client |
| CancelledError swallowed → zombie task | Medium | Explicitly use `except Exception` (not BaseException) in broadcast loop body; let CancelledError propagate to sleep() |
| Concurrent disconnect + broadcast race | Low | ConnectionManager uses put_nowait; writer task's `except Exception: pass` handles closed socket gracefully |

ASVS L1 controls applied: no sensitive data in error logs (snapshots contain only simulation metrics), no auth tokens (local dev), no PII in wire format.
</threat_model>
