<!-- Review feedback that MUST be addressed in this plan revision. -->

HIGH SEVERITY (MUST address):
1. Governor monitoring lifecycle (Codex): Plan must call governor.start_monitoring()
   before dispatch and governor.stop_monitoring() in finally. Without this, governor
   is just a fixed token pool -- no RAM throttling/pause engages.
2. Async-safe app initialization (Codex): create_app_state() calls run_until_complete()
   for Neo4j verify_connectivity, which is UNSAFE inside a running event loop.
   _handle_run() must create AppState BEFORE asyncio.run(), not inside it.

MEDIUM SEVERITY (SHOULD address):
3. Positional alignment risk (both): Index-based pairing of worker_configs with
   decisions is an implicit contract. Add a defensive assertion.
4. Orchestrator model lifecycle dependency (both): No fallback if inject_seed()
   leaves orchestrator loaded. Add ensure_clean_state() before worker load.
5. Schema/bootstrap ownership ambiguity (Codex): Make explicit that CLI handler
   owns schema/cleanup, run_round1() owns seed->dispatch->persist.
6. Round1Result redundancy (Codex): Remove redundant `decisions` field. Derive
   from `agent_decisions` instead.
7. Failure-path cleanup incomplete (Codex): Governor stop and graph close need
   explicit guarantees on all error paths.
8. Rationale sanitization (Codex LOW): Whitespace-normalize and strip control
   characters before printing.
</review_concerns>

    which is UNSAFE because `create_app_state()` calls `asyncio.get_event_loop().run_until_complete()`
    for Neo4j connectivity verification. When called inside `asyncio.run()`, there is already
    a running event loop, and `run_until_complete()` will raise `RuntimeError: This event loop
    is already running`.

    The fix: `_handle_run()` must NOT be async. Instead, create AppState synchronously BEFORE
    `asyncio.run()`, then pass it into an async pipeline function. The structure is:

    ```python
    def _handle_run(rumor: str) -> None:
        """Synchronous CLI handler -- creates AppState BEFORE event loop starts."""
        from alphaswarm.app import create_app_state
        settings = AppSettings()
        brackets = load_bracket_configs()
        personas = generate_personas(brackets)
        app = create_app_state(settings, personas, with_ollama=True, with_neo4j=True)
        # AppState created safely (no running event loop yet)

        assert app.ollama_client is not None
        assert app.model_manager is not None
        assert app.graph_manager is not None

        # NOW start the event loop with the async pipeline
        asyncio.run(_run_pipeline(rumor, settings, app, personas, brackets))

    async def _run_pipeline(
        rumor: str,
        settings: AppSettings,
        app: AppState,
        personas: list[AgentPersona],
        brackets: list[BracketConfig],
    ) -> None:
        """Async pipeline: schema -> round1 -> report -> cleanup."""
        try:
            await app.graph_manager.ensure_schema()
            result = await run_round1(
                rumor=rumor,
                settings=settings,
                ollama_client=app.ollama_client,
                model_manager=app.model_manager,
                graph_manager=app.graph_manager,
                governor=app.governor,
                personas=personas,
            )
            _print_round1_report(result, personas, brackets)
        finally:
            await app.graph_manager.close()
    ```

    This pattern ensures create_app_state() runs BEFORE asyncio.run() starts the event loop,
    avoiding the run_until_complete() conflict. The `ensure_schema()` and `close()` run inside
    the event loop where they belong (they are proper async functions).
