<threat_model>
## Threat Model (ASVS L1)

**Ticker symbol injection:** LLM-extracted ticker values are used in downstream API calls (Phase 17+). The `ExtractedTicker.symbol` field accepts any string from LLM output. Mitigation: In Plan 02, SEC validation acts as an allowlist -- only symbols present in the SEC dataset pass through. In this plan, Pydantic validation ensures the field is a non-empty string, but the actual allowlist check is deferred to the validator callback.

**Malformed JSON from LLM:** The existing 3-tier parse fallback (Tier 1: direct JSON, Tier 2: regex extraction, Tier 3: empty fallback) handles corrupted or unexpected LLM output. Tickers default to empty list on any parse failure -- simulation proceeds safely.

**Integer overflow / relevance bounds:** `relevance: float = Field(ge=0.0, le=1.0)` enforces bounds via Pydantic. Out-of-range values from LLM are rejected at parse time (individual ticker skipped, not crash).

**Denial of service via large ticker array:** LLM could emit hundreds of ticker objects. Mitigation: the top-3 cap truncates after sorting, so at most 3 tickers survive regardless of input size. The parse loop iterates the raw list but each iteration is O(1) Pydantic validation.
</threat_model>

        # Parse tickers (Phase 16: TICK-01)
        dropped: list[dict[str, str]] = []
        raw_tickers = data.get("tickers", [])
        all_tickers: list[ExtractedTicker] = []
        if isinstance(raw_tickers, list):
            for t in raw_tickers:
                try:
                    ticker = ExtractedTicker.model_validate(t)
                    # SEC validation via callback (TICK-02)
                    if ticker_validator and not ticker_validator(ticker.symbol):
                        dropped.append({"symbol": ticker.symbol, "reason": "invalid"})
                        logger.warning("ticker_invalid", symbol=ticker.symbol)
                        continue
                    all_tickers.append(ticker)
                except (ValidationError, TypeError):
                    continue

        # Sort by relevance descending, cap at 3 (TICK-03, per D-05)
        all_tickers.sort(key=lambda t: t.relevance, reverse=True)
        if len(all_tickers) > 3:
            for t in all_tickers[3:]:
                dropped.append({"symbol": t.symbol, "reason": "cap"})
            all_tickers = all_tickers[:3]
