**User vs agent bubble distinction (Claude's Discretion — resolved):**

This alignment + color approach provides clear visual distinction without introducing new color tokens. The accent-tinted user bubble connects visually to the Send button, reinforcing the "your message" affordance.

### App.vue — Modifications

**New state:**

```typescript
const interviewAgentId = ref<string | null>(null)
```

**Event handling:**

```typescript
function onOpenInterview(agentId: string) {
  interviewAgentId.value = agentId
  selectedAgentId.value = null  // close AgentSidebar
}

function onCloseInterview() {
  interviewAgentId.value = null
}
```

**Template — sidebar slot (replaces current single-component Transition):**

```html
<Transition name="sidebar">
  <InterviewPanel
    v-if="interviewAgentId"
    :agentId="interviewAgentId"
    @close="onCloseInterview"
  />
  <AgentSidebar
    v-else-if="selectedAgentId"
    :agentId="selectedAgentId"
    @close="onCloseSidebar"
    @open-interview="onOpenInterview"
  />
</Transition>
```

Note: Vue `<Transition>` requires a single root child — use a wrapping `<div>` with `:key` if needed to satisfy this constraint, OR move to two separate `<Transition>` blocks with `v-if` guards. The executor should choose whichever approach compiles cleanly.

**Graph container width:** The `graph-container--sidebar-open` class should also apply when `interviewAgentId` is set:

```typescript
const sidebarOpen = computed(() => selectedAgentId.value !== null || interviewAgentId.value !== null)
```

## Interaction Contract

### AgentSidebar — Interview Button (D-01, D-02, D-07)

- "Interview Agent NN" button appears at the bottom of AgentSidebar ONLY when `snapshot.phase === 'complete'`
- Click emits `@open-interview` with the current `agentId`
- App.vue handler sets `interviewAgentId` and clears `selectedAgentId` (D-04)
- No new ForceGraph click behavior — existing `@select-agent` is unchanged (D-02)

### InterviewPanel — Conversation Flow (D-05, D-06, D-08)

1. Panel opens with empty chat area
2. User types a question in the textarea, presses Enter or clicks Send
3. User message appended to the message list immediately (optimistic UI)
4. `isLoading = true` — textarea disabled, Send button disabled, "Thinking..." indicator shown
5. `POST /api/interview/{agent_id}` fires with `{ message: str }`
6. On success: agent response appended to message list, `isLoading = false`, "Thinking..." hidden, auto-scroll to bottom
7. On failure: `errorMessage` set, `isLoading = false`, user's message remains in the list (no rollback), user can retry by sending again
8. Chat auto-scrolls to the newest message after each append

### InterviewPanel — Session Persistence (D-05)

- Frontend stores conversation history in reactive state: `Map<string, Message[]>` keyed by `agent_id`
- Closing the panel (X button) does NOT clear history — reopening the same agent resumes the conversation display
- History is cleared on page refresh (no LocalStorage — per deferred decision)
- The `Map` lives in `App.vue` and is passed to `InterviewPanel` as a prop (or provided via inject)

### InterviewPanel — Close Behavior (D-03)

- X button sets `interviewAgentId = null` in App.vue
- Panel slides out with existing sidebar transition (200ms ease-in)
- Returns to empty state — no sidebar is shown
- To open AgentSidebar again, user clicks a graph node (existing behavior)

### InterviewPanel — Error Handling (Claude's Discretion — resolved)

- Network errors: `errorMessage = "Could not reach the server. Try again."`
- HTTP 500: `errorMessage = "Interview failed. Try sending again."`
- HTTP 404 (agent not found): `errorMessage = "Agent not found in simulation data."`
- Error text appears inline below the input row, using `--color-destructive`
- Error auto-clears when the next send request succeeds

### "New Conversation" Button (Claude's Discretion — resolved: NOT included)

A "New conversation" button is NOT included in Phase 35. Rationale:
- The session resets on page refresh (D-05)
- Adding a reset mid-session introduces complexity (must also clear the backend `InterviewEngine` instance) with minimal user benefit
- The backend `InterviewEngine` has a sliding window (`WINDOW_SIZE = 10`) that naturally prevents context overflow
- If needed in the future, this is listed in Deferred Ideas
