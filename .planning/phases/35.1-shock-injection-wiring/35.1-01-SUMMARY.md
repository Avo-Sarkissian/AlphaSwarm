# Phase 35.1 Plan 01: Shock Event Writer Summary

**One-liner:** Neo4j `write_shock_event` writer with UUID4 shock_id, parameterized Cypher, Neo4jWriteError wrapping, and `shock_id_unique` uniqueness constraint — unblocking the B1 shock injection dead feature.

## What Was Built

`GraphStateManager.write_shock_event(cycle_id, shock_text, injected_before_round) -> str` creates a `ShockEvent` node and `(Cycle)-[:HAS_SHOCK]->(ShockEvent)` relationship in Neo4j. This is the write-side counterpart to the existing `read_shock_event` (line 1526) and `read_shock_impact` (line 1569) that previously always returned empty because no writer existed.

## Files Modified

### `src/alphaswarm/graph.py`

- **Lines 60-70 (`SCHEMA_STATEMENTS`):** Appended `shock_id_unique` uniqueness constraint as last entry:
  ```
  "CREATE CONSTRAINT shock_id_unique IF NOT EXISTS FOR (se:ShockEvent) REQUIRE se.shock_id IS UNIQUE"
  ```
- **Lines 1735-1809 (new section):** Added Phase 35.1 section comment + two new methods:
  - `write_shock_event` (async, public): generates UUID4 shock_id, opens session-per-method, delegates to transaction function, wraps `Neo4jError` in `Neo4jWriteError`, logs `shock_length` (not `shock_text`)
  - `_write_shock_event_tx` (static async): executes parameterized Cypher with `MATCH (c:Cycle)` + two `CREATE` statements; all input via `$param` bindings, no f-strings

### `tests/test_graph.py`

- **Lines 1638-1714 (4 new tests):**
  1. `test_write_shock_event_creates_node_and_edge` — verifies `execute_write` called once with correct positional args `(tx_fn, cycle_id, shock_id, shock_text, injected_before_round)`
  2. `test_write_shock_event_returns_uuid4` — verifies return value matches UUID4 regex `^[0-9a-f]{8}-...-4...-[89ab]...-...$`
  3. `test_write_shock_event_wraps_driver_errors` — verifies `Neo4jError` raises `Neo4jWriteError` with correct message and `__cause__`
  4. `test_schema_includes_shock_id_unique_constraint` — synchronous test verifying `SCHEMA_STATEMENTS` contains the constraint
