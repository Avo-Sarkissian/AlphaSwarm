# Phase 35.1 Plan 02: Shock Wiring in run_simulation Summary

**One-liner:** `consume_shock: Callable[[], str | None] | None = None` threaded through `run_simulation` with Round 2 and Round 3 shock gates; `SimulationManager._run` wires `consume_shock=self.consume_shock` closing the B1 dead-feature gap end-to-end.

## What Was Built

Two gaps from BUGFIX-CONTEXT.md section B1 are now closed:

1. **`run_simulation` shock gates** — before Round 2 dispatch and before Round 3 dispatch, `consume_shock()` is called. If it returns non-None text, `effective_rumor_rN` is built as `f"{rumor}\n\n[MARKET SHOCK]: {shock_text}"`, `graph_manager.write_shock_event(cycle_id, shock_text, injected_before_round=N)` is awaited, and a structlog `shock_injected` event is emitted with `shock_length` only. The base `rumor` string is never mutated.

2. **`SimulationManager._run` wiring** — `consume_shock=self.consume_shock` added as the final kwarg in the `run_simulation(...)` call, completing the call chain: `POST /api/simulate/shock` → `inject_shock` → `_pending_shock` → `consume_shock` → `run_simulation` → Round 2/3 `user_message` augmentation → `graph_manager.write_shock_event` → `ShockEvent` node in Neo4j.

## Threat Mitigations Applied

| Threat ID | Mitigation |
|-----------|------------|
| T-35.1-05 | `shock_text` wrapped with `[MARKET SHOCK]:` delimiter in the `f"{rumor}\n\n[MARKET SHOCK]: {shock_text}"` pattern — framed input for LLM prompt |
| T-35.1-06 | `write_shock_event` called with `injected_before_round` matching the consuming round — full audit trail in Neo4j |
| T-35.1-07 | `consume_shock` drains `_pending_shock` on first call; called AT MOST ONCE per round — at most one ShockEvent per round |
| T-35.1-08 | Accepted — `run_simulation` is linear; no concurrent `consume_shock` calls within a single cycle |
| T-35.1-09 | `shock_injected` structlog emits `shock_length=len(shock_text)` only, never `shock_text` content |

## End-to-End Call Chain (Now Live)

```
ShockDrawer (UI)
  -> POST /api/simulate/shock
  -> SimulationManager.inject_shock()
  -> _pending_shock (stored)
  -> SimulationManager.consume_shock() [called by run_simulation]
  -> run_simulation Round 2/3 shock gate
  -> effective_rumor_rN = f"{rumor}\n\n[MARKET SHOCK]: {shock_text}"
  -> dispatch_wave(user_message=effective_rumor_rN)  [100 LLM agents see shock]
  -> graph_manager.write_shock_event(cycle_id, shock_text, injected_before_round=N)
  -> ShockEvent node in Neo4j
  -> read_shock_event / read_shock_impact (Phase 27 reporting)
```
