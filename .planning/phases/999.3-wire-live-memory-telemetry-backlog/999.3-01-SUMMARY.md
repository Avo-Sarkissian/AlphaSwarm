# Phase 999.3 Plan 01: Wire Live Memory% Telemetry Summary

**One-liner:** Broadcaster-side psutil overlay writes live host RAM% into `governor_metrics.memory_percent` on every 5Hz tick (IDLE + replay + active), gated by a D-02 phase guard in the TUI footer so IDLE never displays RAM%.

## What Was Built

Fixed KR-41.1-04: the WebSocket state frame's `governor_metrics.memory_percent` was always 0 because the governor only emits metrics on state transitions and only runs during active simulations. The fix reads `psutil.virtual_memory().percent` once per `snapshot_to_json` call (200ms/5Hz tick) and overlays it onto the wire frame for both live and replay branches, synthesizing a minimal `governor_metrics` dict when the governor hasn't emitted one.

## Decisions Implemented

- **D-01:** Replay mode gets live host RAM% overlay — same KPI label means "current host RAM" everywhere. Replay branch now calls `_overlay_memory(d)` before `json.dumps`.
- **D-02:** TUI footer IDLE phase guard — `if snapshot.phase == SimulationPhase.IDLE: self._render_idle(); return` added at the top of `update_from_snapshot`, before the `gm is None` check. The TUI uses `state_store.snapshot()` directly (not the wire frame), so `governor_metrics` stays None in TUI during IDLE — but the guard is defensive against future code paths.
