<objective>
Wire the real `YFinanceMarketDataProvider` and `RSSNewsProvider` into both the web lifespan
(D-10) and the CLI `_run_pipeline` (D-11) so simulations running via the web SPA or `alphaswarm
run` command actually see grounded market data. Extend `AppState` with provider fields so the
web path can share state across SimulationManager and future consumers without ad-hoc
attribute lookups on `app.state`. Add TWO integration tests:

1. A **forwarding test** (honestly named) proving `run_simulation → run_round1` receives
   a formatted `market_context` string (this is the run_simulation-level contract).
2. A **dispatch-depth test** proving `run_round1 → dispatch_wave → AgentWorker.infer`
   delivers the `market_context` system message at the correct index in the messages list
   (this closes the "does Round 1 actually get the string?" gap).

Purpose: Converts the behind-the-wall infrastructure from Plans 01 and 02 into a
user-facing feature — users launching via `uvicorn alphaswarm.web.app:app` or
`uv run alphaswarm run "<rumor>"` get context-grounded Round 1 prompts for all 100 agents
without any flag changes. Closes the deployment-surface half of INGEST-03 and SIM-04.

**Tests revised per REVIEWS feedback (2026-04-19):**
- Lifespan wiring test uses an extended `_make_test_app()` (NOT production `create_app()`)
  to preserve the existing isolation pattern (AppSettings(_env_file=None), with_ollama=False,
  with_neo4j=False). This prevents Neo4j/Ollama/.env side effects from leaking into CI.
- What was originally called "end-to-end" is split into two honest tests: a `forwards_*`
  test that patches `run_round1` (proves run_simulation assembly contract) and a
  `through_dispatch_wave` test that patches `AgentWorker.infer` and runs the real
  `dispatch_wave` helper (proves the system message reaches the worker). Together these
  close the plumbing chain at both layers without making misleading claims.

Output: AppState schema extended, two wiring call sites updated, four tests added
(lifespan wiring, CLI construction, forwarding to run_round1, dispatch-depth to worker).
</objective>
