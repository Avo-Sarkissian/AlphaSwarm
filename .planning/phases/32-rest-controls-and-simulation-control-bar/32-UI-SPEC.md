## Component Inventory

### ControlBar.vue

Persistent top strip, always visible regardless of simulation state (D-01).

| Property | Value |
|----------|-------|
| Width | 100vw |
| Height | 48px fixed (`--space-2xl` equivalent; add `--control-bar-height: 48px` token) |
| Background | `#1a1d27` (`--color-bg-secondary`) |
| Border bottom | 1px solid `#374151` (`--color-border`) |
| Padding | 0 16px (`--space-md` horizontal) |
| Z-index | 20 (above sidebar z-index 10, above graph) |
| Layout | `display: flex; align-items: center; gap: 8px` |
| Position | `position: relative` (not fixed -- flex child of app-root) |

#### Idle State (phase === 'idle' OR phase === 'complete')

Left-to-right layout within the 48px strip:

1. **Seed textarea** -- 1 line height, `flex: 1` to fill available width, max-height 32px, Body 14px/400, placeholder "Enter a seed rumor...", vertically centered
2. **Start button** -- Label 12px/600, text "Start Simulation", height 32px, padding 0 16px, border-radius 4px, accent background `#3b82f6`

When `phase === 'complete'`, both elements are enabled (user can start a new simulation).

#### Active State (phase !== 'idle' AND phase !== 'complete')

Left-to-right layout:

1. **Seed textarea** -- same element, now `disabled`, opacity 0.5, cursor `not-allowed`
2. **Start button** -- `disabled`, background `#374151`, text `#6b7280`, cursor `not-allowed`
3. **Phase label** -- Label 12px/400, text `#9ca3af`, e.g. "Round 2 / 3", margin-left auto (pushes to right side), `white-space: nowrap`
4. **Stop button** -- Label 12px/600, text "Stop", height 32px, padding 0 16px, border-radius 4px, destructive background `#ef4444`
5. **"+Inject Shock" button** -- Label 12px/400, text "+Inject Shock", height 32px, padding 0 16px, border-radius 4px, ghost style (transparent bg, accent border + text)

Phase label text mapping:

| `snapshot.phase` | Label Text |
|------------------|------------|
| `seeding` | "Seeding..." |
| `round_1` | "Round 1 / 3" |
| `round_2` | "Round 2 / 3" |
| `round_3` | "Round 3 / 3" |
| `replay` | "Replay" |

#### Disabled Logic

| Element | Disabled When |
|---------|---------------|
| Seed textarea | `phase !== 'idle' && phase !== 'complete'` |
| Start button | `phase !== 'idle' && phase !== 'complete'`, OR seed textarea is empty |
| Stop button | Never disabled while visible (only rendered in active state) |
| +Inject Shock button | Never disabled while visible (only rendered in active state) |

### App.vue Modifications

The existing `app-root` layout changes from `position: relative` single-child to a flex column:

```
app-root (flex column, 100vh)
  +-- ControlBar (flex-shrink: 0, height: 48px)
  +-- ShockDrawer (flex-shrink: 0, conditional, max-height: 160px)
  +-- main-content (flex: 1, overflow: hidden)
       +-- empty-state (when idle) OR graph-container (when active)
  +-- AgentSidebar (fixed position, unchanged)
  +-- connection-error (fixed position, unchanged)
```

| Property | Value |
|----------|-------|
| Layout | `display: flex; flex-direction: column; height: 100vh` |
| Main content area | `flex: 1; overflow: hidden; position: relative` |

The graph container and empty state render inside the main content area, which automatically fills remaining vertical space after the control bar (and optional shock drawer). This resolves Pitfall 6 from RESEARCH.md without `calc()` hacks.

The existing `isIdle` computed changes: idle shows empty state in the main content area, but the control bar is always visible above it with the seed input ready.

## Interaction States

### Control Bar -- Idle

| Element | Specification |
|---------|---------------|
| Seed textarea | Enabled, empty, placeholder "Enter a seed rumor..." |
| Start button | Disabled until textarea has content (empty string check with trim) |
| Phase label | Not rendered |
| Stop button | Not rendered |
| +Inject Shock | Not rendered |

### Control Bar -- Active

| Element | Specification |
|---------|---------------|
| Seed textarea | Disabled, retains the seed text that was submitted, opacity 0.5 |
| Start button | Disabled, greyed out (#374151 bg, #6b7280 text) |
| Phase label | Shows current phase, updates in real time from WebSocket snapshot |
| Stop button | Enabled, destructive red |
| +Inject Shock | Enabled, ghost accent style |

### Control Bar -- Complete

Same as Idle. Seed textarea is cleared and re-enabled. Start button awaits new input.

### Shock Drawer -- Submit Flow

1. User clicks "+Inject Shock" in control bar
2. Shock drawer slides down (200ms ease-out)
3. Textarea auto-focuses
4. User types shock text and clicks "Inject Shock"
5. Submit button shows "Injecting..." (disabled)
6. `POST /api/simulate/shock` fires with `{shock_text: "..."}`
7. On 200 response: drawer slides up (150ms ease-in), shock text cleared
8. On 409 response: inline error appears below textarea, drawer stays open
9. User can click Cancel at any time to close without submitting

### Stop Flow

1. User clicks "Stop" in control bar
2. `POST /api/simulate/stop` fires
3. On 200 response: simulation cancels, phase transitions to `idle` or `complete` via WebSocket
4. On 409 response: no-op (simulation already stopped between click and request)

### Start Flow

1. User types seed rumor into textarea
2. Start button enables when textarea is non-empty (trimmed)
3. User clicks "Start Simulation"
4. Start button shows "Starting..." (disabled) for the duration of the fetch
5. `POST /api/simulate/start` fires with `{seed: "..."}`
6. On 202 response: control bar transitions to active state as WebSocket snapshot phase changes from `idle` to `seeding`
7. On 409 response: no-op (simulation already running -- should not happen if UI state is correct, but guards against race conditions)

### Empty State Update

The Phase 31 empty state ("Waiting for Simulation") now renders below the control bar. The control bar's seed textarea serves as the primary CTA, replacing the passive "Start a simulation to see 100 agents deliberate in real time." instruction with an actionable input. The empty state heading and body copy remain as ambient context below the control bar.
