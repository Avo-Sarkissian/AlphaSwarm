## Decisions Made

- **Single routing path (Codex HIGH fix):** LifecycleOverlay is the only overlay routing entry point. app_v2.jsx no longer contains any `if (ds === 'idle')` / `if (ds === 'seeding')` logic — all precedence lives in states.jsx.
- **Sibling rendering over conditional block:** `<LifecycleOverlay>` and the live-graph elements (Viz, ConsensusRing, Legend, Tooltip) are siblings. LifecycleOverlay returns `null` when no overlay should show, and the `.state-overlay` class uses `position: absolute` to cover the canvas when an overlay is active.
- **demoState removal:** The constant was always `'live'` and had no callers or storybook fixtures that toggled it — confirmed safe to remove.

## Follow-up Note (Codex LOW Review)

`TaskBannerProps.phase` is currently typed as `phase?: string` (optional). Now that app_v2.jsx is the only caller and always passes the live `phase` value, this should be promoted to `phase: string` (required) in a follow-up clean-up pass. Can be folded into the next chrome-strip phase or filed as a Quick fix.
