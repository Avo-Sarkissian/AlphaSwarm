<threats>
  <threat id="T-01">
    <description>Jinja2 template crashes with UndefinedError or TypeError if `data` dict is missing expected keys (e.g., `data.pivot_agents` not present when pivot_count=0).</description>
    <severity>MEDIUM</severity>
    <mitigation>Use `data.get("pivot_agents", [])` and `data.get("notable_held_firm_agents", [])` in template conditionals. Use `| default([])` Jinja2 filter as fallback. Always check `{% if data.pivot_agents %}` before rendering the pivot list table.</mitigation>
  </threat>
  <threat id="T-02">
    <description>CLI pre-seeding block calls read_shock_event on every `report` invocation, adding a Neo4j round-trip even for no-shock cycles. If Neo4j is down, report generation fails entirely.</description>
    <severity>LOW</severity>
    <mitigation>Wrap the shock pre-seeding block in try/except Exception (mirroring the portfolio block). On exception, log `log.warning("shock_impact_skipped", reason=str(exc))` and continue without adding the observation. The report generates without the shock section — a safe degraded result.</mitigation>
  </threat>
  <threat id="T-03">
    <description>Adding shock_impact to SECTION_ORDER when the Jinja2 template file does not yet exist causes a TemplateNotFound error at assemble() time, crashing the report for all cycles including no-shock cycles.</description>
    <severity>HIGH</severity>
    <mitigation>BLOCKED: Create 11_shock_impact.j2 in Task 1 BEFORE modifying TOOL_TO_TEMPLATE and SECTION_ORDER in Task 2. The template file must exist on disk before any code references it.</mitigation>
  </threat>
  <threat id="T-04">
    <description>Shock text injected by a user could contain Jinja2-interpretable syntax (`{{`, `{%`) which, if autoescape is disabled in the report Environment, would execute template code.</description>
    <severity>HIGH</severity>
    <mitigation>The existing ReportAssembler Environment sets `autoescape=False` (markdown output). Mitigate by piping shock_text through the `| e` filter in the template or escaping `{` as `{{ '{' }}` — but since this is a local-only tool with no external users, the primary mitigation is that SHOCK_TEXT_MAX_LEN=4096 (already enforced in Phase 26 StateStore) prevents extreme payloads. Add a note in the template: pass shock_text through `{{ data.shock_text | replace("{", "&#123;") | replace("}", "&#125;") }}` only in the HTML template where it is inserted into HTML context. In the markdown template, use `{{ data.shock_text }}` directly (markdown renderers do not execute Jinja2). This is ASVS Level 1 compliant for a local tool.</mitigation>
  </threat>
</threats>
