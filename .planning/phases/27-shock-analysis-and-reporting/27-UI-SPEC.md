# Phase 27 — UI Design Contract

> Visual and interaction contract for the BracketPanel delta mode (TUI) and the `11_shock_impact.j2` report section (markdown + HTML). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope

Phase 27 has **two delivery surfaces**, both locked to AlphaSwarm's existing dark-terminal aesthetic:

1. **BracketPanel delta mode** — an in-place state transition on the existing `BracketPanel` widget (`src/alphaswarm/tui.py:314`). Post-simulation, when a ShockEvent exists for the current cycle, the live BUY/SELL/HOLD bar per bracket is replaced with a compact **pre → post delta bar** showing the shock-induced signal shift. No new widgets, no layout changes, no new color tokens.
2. **Report section 11 — Shock Impact Analysis** — a new Jinja2 template `11_shock_impact.j2` rendered into both the markdown report (via `ReportAssembler`) and the HTML report (via `assemble_html()`). Appended after `portfolio_impact` in `SECTION_ORDER`. Rendered only when `read_shock_event()` returns non-None for the cycle.

**Out of scope for this UI contract:**
- `read_shock_event` / `read_shock_impact` Cypher query authoring (non-visual; RESEARCH.md Pattern 2)
- CLI pre-seeded `ToolObservation` wiring (non-visual; RESEARCH.md Pattern 1)
- `_poll_snapshot` delta-mode latch and `run_worker` invocation (non-visual interaction glue; RESEARCH.md Pattern 3)
- SHOCK-04 / SHOCK-05 REQUIREMENTS.md traceability entry (documentation-only)

## Layout and Composition

### TUI BracketPanel — delta mode

Delta mode replaces the live-bar rendering with a **compact delta-only bar** (Claude's Discretion per RESEARCH.md Pattern 3, recommended variant). It preserves the existing panel geometry so the transition is a **pure data swap** — no reflow, no flicker.

```
┌────────────────────────────────────────┐
│ Brackets                               │  <- bold #4FC3F7 (unchanged)
│ [DELTA · Shock before Round 2]         │  <- #4FC3F7 sub-header (NEW, single line)
│                                        │
│ Quants         BUY  ▲ [██████░░░░░░░░░] +30%  │  <- #66BB6A fill, bold delta
│ Degens         SELL ▼ [████░░░░░░░░░░░] -15%  │  <- #EF5350 fill, bold delta
│ Sovereigns     HOLD · [░░░░░░░░░░░░░░░]  0%   │  <- #78909C fill, regular
│ Macro          BUY  ▲ [██░░░░░░░░░░░░░] +08%  │
│ Suits          SELL ▼ [████████░░░░░░░] -40%  │
│ Insiders       BUY  ▲ [█░░░░░░░░░░░░░░] +05%  │
│ Agents         HOLD · [░░░░░░░░░░░░░░░]  0%   │
│ Doom-Posters   SELL ▼ [███████████░░░░] -55%  │
│ Policy Wonks   BUY  ▲ [████░░░░░░░░░░░] +20%  │
│ Whales         HOLD · [░░░░░░░░░░░░░░░]  0%   │
└────────────────────────────────────────┘
```

**Row anatomy (per bracket):**

```
{display_name:<14}  {dominant_signal:<4} {arrow} [{fill}{empty}] {delta:+d}%
│                   │                    │      │                │
│                   │                    │      │                └─ bold, colored by sign
│                   │                    │      └─ bar: fill = abs(delta) / 100 * BAR_WIDTH
│                   │                    └─ ▲ (green) / ▼ (red) / · (gray)
│                   └─ BUY / SELL / HOLD (dominant POST-shock signal)
└─ 14 cells, #78909C
```

**Widget tree (unchanged — same `BracketPanel` class, new render branch):**

```python
BracketPanel(Widget):
    # Existing state
    _summaries: tuple[BracketSummary, ...]
    # New state (delta mode)
    _delta_mode: bool = False
    _delta_data: dict | None = None   # payload from GraphStateManager.read_shock_impact()

    def render(self) -> Text:
        if self._delta_mode and self._delta_data:
            return self._render_delta()  # NEW branch
        return self._render_live()       # EXISTING (extracted from current render)
```

**Activation flow (non-visual, for executor reference):**

1. Dashboard `_poll_snapshot()` observes `snapshot.phase == SimulationPhase.COMPLETE`
2. Latch flag `_bracket_panel_delta_active` is False (first-time trigger)
3. Dashboard `run_worker(self._activate_bracket_delta_mode(self._cycle_id))` — async worker
4. Worker calls `gm.read_shock_event(cycle_id)`; if None, worker exits silently (live mode persists)
5. Worker calls `gm.read_shock_impact(cycle_id)`; on success calls `bracket_panel.enable_delta_mode(data)`
6. `enable_delta_mode()` sets `_delta_mode=True`, `_delta_data=data`, calls `self.refresh()`
7. Latch flag is set; subsequent 200ms ticks do not re-trigger the worker

**Persistence:** Delta mode persists until the user quits the app. No auto-revert. This matches the "freeze state at COMPLETE" convention from Phase 10.

**Graceful degradation:** If `_delta_data` is present but malformed (missing bracket, divide-by-zero), `_render_delta()` falls back to `_render_live()` silently and logs a `structlog` warning via `component="tui"`.

## Interaction Contract

### TUI BracketPanel delta mode

| Interaction | Trigger | Effect |
|-------------|---------|--------|
| Enter delta mode | `_poll_snapshot()` sees `phase == COMPLETE` + `read_shock_event()` returns non-None | `bracket_panel.enable_delta_mode(delta_data)` → `render()` branches to `_render_delta()` |
| Re-render on refresh | `bracket_panel.refresh()` call post-`enable_delta_mode()` | Rich `Text` re-emitted with delta rows; panel remains 40 cells wide, same position |
| Skip activation (no shock) | `read_shock_event()` returns None | No-op; live mode persists (final-round summaries shown) |
| Stay in delta mode | User continues to use TUI post-simulation (interview, quit) | Delta mode persists until app exit — no revert mechanism |
| Interview click | User clicks an AgentCell in the 10x10 grid (Phase 14) | Unchanged — interview overlay pushes on top of dashboard, BracketPanel delta mode remains visible when interview dismisses |

**Key bindings added:** none. Delta mode is a passive state change — no new user-triggered action.
**Focus order:** unchanged. BracketPanel is non-focusable (display-only widget).
**Mouse affordance:** none on the panel itself.

### Report section 11 (static document)

The report is a static document — there are no interactive behaviors to declare. HTML report links already anchor via the Phase 24 `id="shock-impact"` convention.

## State Contract

### BracketPanel (TUI) state machine

| State | Visual | Trigger → Next |
|-------|--------|----------------|
| Awaiting data | `Awaiting data...` text in `#78909C` | Trigger: initial mount with empty `_summaries`. Next: first snapshot → Live |
| Live (rounds 1-3) | 10 bracket rows, each with dominant-signal bar, percentage label | Trigger: `update_summaries()` with non-empty tuple. Next: `phase==COMPLETE` + shock detected → Delta |
| Live (final) | Same as Live, showing Round 3 summaries | Trigger: `phase==COMPLETE`, no shock. Next: terminal state (persists until quit) |
| Delta | Sub-header + 10 delta-bar rows with pre→post deltas | Trigger: `enable_delta_mode(data)` called from worker. Next: terminal state (persists until quit) |
| Fallback | Reverts to Live rendering with `structlog` warning | Trigger: `_delta_data` malformed. Next: Live (final) |

**No loading state** on the delta transition. The async graph read happens in a `run_worker` — the panel continues showing Live (final) until the worker completes and calls `enable_delta_mode()`. The swap is atomic on the next `refresh()`.

**No error state** on the panel. Graph read failures bubble to `structlog` only; the panel falls back to Live rendering.

### Report section 11 state machine

| State | Visual | Trigger |
|-------|--------|---------|
| Absent | Section does not appear in markdown or HTML | `read_shock_event()` returned None OR no `shock_impact` ToolObservation in CLI handler |
| Rendered | Full section with 5 sub-blocks (header, bracket shift, pivot summary, pivot list, notable held-firm) | `shock_impact` ToolObservation present with non-empty `data` |
| Partial (graceful) | Header + bracket shift only; "No agents pivoted this round" where pivot list would be | `data.pivot_agents` is empty but `data.bracket_deltas` is non-empty — edge case for perfectly-held-firm cycles |

**No error state** in the report. If `read_shock_impact()` raises, the CLI handler (outside this UI contract) logs and skips pre-seeding — the section becomes Absent.

## Accessibility

### TUI delta mode

- **Screen reader / terminal emulator support:** Rich `Text` spans render into the terminal text buffer in source order. Screen readers reading Terminal.app will read `Brackets`, the delta sub-header, then each row in `{bracket} {signal} {delta}` order. The arrow glyphs (`▲`/`▼`/`·`) convey direction visually but are redundantly encoded in the signed delta text (`+30%` / `-15%` / ` 0%`), so non-visual consumers lose no information.
- **Keyboard-only operation:** No user action required to enter delta mode — it activates automatically post-simulation. Dashboard keyboard bindings (q to quit, click to interview) remain unchanged.
- **Contrast (WCAG AA on terminal rendering):**
  - `#66BB6A` on `#1E1E1E` (BUY fill on panel): contrast ratio 7.8:1 — **PASS AAA**
  - `#EF5350` on `#1E1E1E` (SELL fill on panel): contrast ratio 5.1:1 — **PASS AA normal text**
  - `#78909C` on `#1E1E1E` (HOLD fill on panel): contrast ratio 4.6:1 — **PASS AA normal text**
  - `#E0E0E0` on `#1E1E1E` (bar empty + row body text): contrast ratio 12.3:1 — **PASS AAA**
  - `#4FC3F7` on `#1E1E1E` (sub-header): contrast ratio 8.9:1 — **PASS AAA**
- **Color independence:** signal identity is encoded redundantly (bar color + signal text `BUY`/`SELL`/`HOLD` + arrow glyph + signed delta number), so color-blind users lose no information.

### Report section 11 (markdown + HTML)

- **Markdown renderers:** section headings (`## Shock Impact Analysis`, `### Bracket Signal Shift`) use standard heading levels; screen readers and outline tools will pick them up. Tables use native markdown pipe syntax.
- **HTML contrast:** all text colors inherit the Phase 24 dark theme (already WCAG AA verified in Phase 24 UI-SPEC).
- **Table semantics:** the HTML rendering wraps tables in `<table>` with `<thead>`/`<tbody>` (Phase 24 convention); screen readers announce headers per cell.
- **Keyboard-only operation:** report is static; no interactive elements.
