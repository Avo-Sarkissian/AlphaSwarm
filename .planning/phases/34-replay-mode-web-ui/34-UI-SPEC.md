## Interaction Contract

### ControlBar вҖ” Idle State Addition

- "Replay" button added to idle strip alongside "Start Simulation"
- Styling: `background: transparent`, `border: 1px solid --color-border`, `color: --color-text-secondary`, 32px height вҖ” ghost variant matching the ShockDrawer `+Inject Shock` button style
- Position: immediately after the Start Simulation button (right side of seed textarea row)
- Click handler: sets `showCyclePicker = true` in App.vue, which mounts `<CyclePicker>`

### CyclePicker вҖ” Fetch Behavior

- Fetch `GET /api/replay/cycles` fires **on modal open** (not eagerly at app load вҖ” per D-02)
- Show "Loading cycles..." during fetch
- On success with 0 results: show empty state copy
- On fetch error (network or 500): show error state copy with a "Retry" link (inline text button, `color: --color-accent`)
- Backdrop click dismisses the modal (same as Close Picker)
- Escape key dismisses the modal

### Replay Start Flow

1. User selects a cycle row (radio selection, row highlight)
2. "Start Replay" button becomes enabled
3. User clicks "Start Replay"
4. Button label changes to "Starting..." during the POST вҖ” button disabled
5. On `POST /api/replay/start/{cycle_id}` success: modal closes, ControlBar switches to replay strip (driven by `snapshot.phase === 'replay'` WebSocket update)
6. On error (404 cycle not found): modal stays open, show inline error below the list: "Cycle not found in Neo4j. Select a different cycle."

### Round Stepping

- `[в–¶ Next]` calls `POST /api/replay/advance`
- During request: button label "Advancing..." and disabled
- On success: WebSocket broadcasts new snapshot вҖ” force graph node colors update (existing color-watch in ForceGraph.vue handles this automatically)
- At Round 3 / 3: `[в–¶ Next]` is permanently disabled, label remains `в–¶ Next` (no text change)
- No confirmation required to step вҖ” action is non-destructive

### Exit Replay

- `[вң• Exit]` calls `POST /api/replay/stop`
- No confirmation dialog вҖ” exit is safe and reversible (user can open CyclePicker again)
- On success: `snapshot.phase` returns to `'idle'` via WebSocket, ControlBar reverts to seed/start strip
- Force graph node states are NOT cleared on exit вҖ” they remain visible from the last replay round until a new simulation/replay begins (consistent with existing `phase === 'idle'` behavior showing the empty state overlay in App.vue)

### Force Graph Behavior in Replay Mode

- Node colors update from `snapshot.agent_states` on each round advance вҖ” existing `ForceGraph.vue` color-watch handles this with zero changes
- Node click вҶ’ AgentSidebar flow unchanged вҖ” works identically in replay mode
- Edges: clear on `phase === 'replay'` start (same as `phase === 'seeding'` clear logic already in ForceGraph), then fetch edges per round as normal using `GET /api/edges/current?round=N`
