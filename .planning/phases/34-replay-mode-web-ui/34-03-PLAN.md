  <how-to-verify>
Open browser to http://localhost:8000

**Test 1 -- Idle state Replay button:**
1. Verify a "Replay" button is visible next to "Start Simulation" when idle
2. Verify the Replay button has ghost styling (transparent background, border)

**Test 2 -- Cycle picker modal:**
3. Click the "Replay" button
4. Verify a modal opens with title "Select a Cycle to Replay"
5. If Neo4j has completed cycles: verify cycle rows appear with truncated cycle ID, date, and seed rumor preview
6. If no cycles: verify "No completed cycles found." message
7. Test backdrop click dismisses the modal
8. Test Escape key dismisses the modal
9. Verify "Start Replay" button is disabled until a cycle is selected

**Test 3 -- Replay start:**
10. Select a cycle (radio button)
11. Click "Start Replay" -- button should show "Starting..."
12. Modal should close, ControlBar should switch to replay strip
13. Verify REPLAY badge appears with amber/orange background (#f59e0b)
14. Verify "Round 1 / 3" indicator is visible
15. Verify force graph nodes appear with colors from stored Round 1 decisions

**Test 4 -- Round stepping:**
16. Click "Next" button -- should show "Advancing..."
17. Verify "Round 2 / 3" appears after advance
18. Verify force graph nodes update colors for Round 2
19. Click "Next" again -- verify "Round 3 / 3"
20. Verify "Next" button is disabled at Round 3/3 (grayed out, not clickable)

**Test 5 -- Force graph interaction in replay:**
21. Click any agent node in the force graph
22. Verify AgentSidebar opens with agent details (same as live mode)
23. Close the sidebar

**Test 6 -- Exit replay:**
24. Click "Exit" button
25. Verify ControlBar returns to idle state (seed textarea + Start Simulation + Replay buttons)
26. Verify force graph returns to empty/idle state

**Test 7 -- Visual distinction:**
27. Confirm the REPLAY badge makes replay mode unmistakable vs live simulation
28. Confirm the replay strip uses the single-row footprint (same height as live control bar)
