Fix D-03: extend `OllamaClient.generate()` to accept and forward a `system` parameter.
Today, `simulation.py:1316` calls `client.generate(..., system=...)` but `generate()`'s
signature (`ollama_client.py:112`) only declares `model, prompt, format, keep_alive, options`.
Every call raises `TypeError`, swallowed by the bare `except Exception` at
`simulation.py:1325`, producing zero per-decision narratives. The interview context that
depends on these narratives is silently degraded.

Decision (LOCKED): Keep the high-level `system + prompt` call style (more readable than
manually concatenating into one prompt). Symmetric with `chat()` which already supports
system messages. Add `system: str | None = None` to `generate()` and forward it.

Output: Fixed signature + unit test confirming forwarding + regression test that
`_generate_decision_narratives()` produces non-zero output for a single-agent fixture.
