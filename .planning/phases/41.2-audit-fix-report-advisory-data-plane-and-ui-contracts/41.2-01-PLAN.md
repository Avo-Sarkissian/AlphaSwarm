<objective>
Fix D-01: rewrite the 6 GraphStateManager Cypher read methods so they compare against the
actual lowercase signal/flip values written by `_batch_write_decisions_tx` and
`_batch_write_episodes_tx`. Today every read returns zero counts because the Cypher
compares against uppercase string literals (`'BUY'`, `'SELL'`, `'HOLD'`, `'NONE'`) while the
writer stores lowercase enum values (`'buy'`, `'sell'`, `'hold'`, `'none'`). This silently
corrupts the v6.0 advisory deliverable.

Purpose: Restore the data plane that downstream report assembly + advisory synthesis
(`alphaswarm/advisory/engine.py:55-61`) rely on. Locked by D-01.

Output: Patched `graph.py` reads + a regression test that proves non-zero counts on a
synthetic mixed-signal cycle.
</objective>
