## Decisions Made

- **Signature insertion point: `system` between `prompt` and `format`.** Preserves caller compatibility for existing positional-arg calls (verified — `simulation.py:1316`, the only existing call in this codebase, uses kwargs; no positional `format=` callers exist for `generate`). Matches the conceptual ordering "what the model sees" (model → user prompt → system framing).
- **Real-wrapper instantiation in regression test.** Test 2 constructs a real `OllamaClient` and mocks only `_client.generate` rather than monkey-patching `OllamaClient.generate` itself. Without this, RED would not fire (a Mock on the wrapper accepts any kwargs). This pattern is documented in `patterns-established` for future kwarg-forwarding bug regressions.

## Deviations from Plan

None - plan executed exactly as written.

The plan's Task 1 sample test code suggested `unittest.mock.patch.object(client, "_client") as mock_inner; mock_inner.generate = fake_underlying`, but explicitly invited the executor to "match [the existing test file's] style" if a different fake-transport convention existed. The existing test file uses the `mock_async_client` fixture (already defined with `client.generate = AsyncMock(...)`) and `ollama_client` fixture wired together via `client._client = mock_async_client`. The implementation reuses these fixtures rather than introducing `patch.object`, which is functionally equivalent and consistent with the file's style. This is style alignment, not a deviation.

## Issues Encountered

**1. Initial Test 2 implementation passed at RED unexpectedly.**

- **Symptom:** First draft of `test_generate_decision_narratives_produces_output` reused the existing `mock_ollama_client` fixture (`MagicMock` with `client.generate = AsyncMock(...)`). With this mock, the wrapper's broken signature is bypassed entirely, so the test passed even before the fix.

This exercises the real wrapper signature, so pre-fix the call raises `TypeError` (swallowed by `simulation.py:1325`'s bare `except Exception`) → 0 narratives → assertion fails → RED. Post-fix, the kwarg flows through and the assertion succeeds → GREEN.
- **Verified:** Task 1 commit `8b1cac2` shows both tests failing; Task 2 commit `d4a2c9b` flips both to passing.
