<objective>
Fix D-02: unify ReACT tool names so the prompt's user-facing contract matches every
downstream registry. Today the prompt advertises `consensus_summary` and `signal_flips`,
but the CLI registry, web registry, and TOOL_TO_TEMPLATE all use `bracket_summary` and
`signal_flip_analysis`. The model dispatches by prompt name, so every report run logs
`ERROR: Unknown tool 'consensus_summary'` (handled but degrades the report).

Decision (LOCKED): the prompt is the user-facing contract — keep prompt names. Rename
two keys in three registries + add an invariant test so this can never silently drift again.

Purpose: Make `alphaswarm.cli report` and `POST /api/report/{id}/generate` succeed
without "unknown tool" warnings, regardless of which order the model calls tools.

Output: Three registries renamed + one invariant test that reads `REACT_SYSTEM_PROMPT`
and asserts every advertised name is present in every registry.
</objective>
