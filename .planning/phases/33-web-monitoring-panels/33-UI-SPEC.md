## Layout Contract

### Overall Structure (D-01, D-02, D-03)

```
+---------------------------------------------------------------+
| ControlBar (48px, --control-bar-height)                       |
+---------------------------------------------------------------+
|                                                               |
|  ForceGraph (flex: 1, takes remaining vertical space)         |
|                                                               |
+---------------------------------------------------------------+
| Panel Strip (232px, --panel-strip-height)                     |
| +----------------------------+------------------------------+ |
| | BracketPanel (50% width)   | RationaleFeed (50% width)    | |
| | D3 stacked bars            | Animated entry list           | |
| +----------------------------+------------------------------+ |
+---------------------------------------------------------------+
```

- `.main-content` changes from single-child to flex column: `display: flex; flex-direction: column;`
- Graph container: `flex: 1; min-height: 0;` (shrinks to accommodate panel strip)
- Panel strip: `height: var(--panel-strip-height); flex-shrink: 0;`
- Panel strip background: `var(--color-bg-secondary)`
- Panel strip top border: `1px solid var(--color-border)`
- Panel strip internal layout: `display: flex; gap: 0;` with a `1px solid var(--color-border)` vertical divider between panels
- Each panel half: `flex: 1; overflow: hidden; padding: var(--space-md);`
- Panels visible only when `!isIdle` (same condition as force graph) -- no placeholder UI in idle state (D-03)

### Sidebar Interaction

When AgentSidebar is open (280px right panel), the panel strip width shrinks alongside the graph. The existing `graph-container--sidebar-open` pattern (`width: calc(100vw - var(--sidebar-width))`) extends to the panel strip as well.

## Component Inventory

### BracketPanel.vue (NEW)

**Purpose:** D3 SVG stacked horizontal proportion bars showing buy/sell/hold distribution per bracket.

**Data source:** `inject('snapshot')` -- reads `snapshot.bracket_summaries: BracketSummary[]`

**Visual specification:**

| Element | Spec |
|---------|------|
| SVG sizing | `viewBox`-based, width 100% of container, height auto-calculated from bracket count |
| Bar count | 10 bars (one per bracket), always rendered even when counts are zero (D-05) |
| Bar height | 20px per bar |
| Bar gap | 4px (`--space-xs`) between bars |
| Bar fill | 3 stacked `<rect>` segments per bar: buy (`#22c55e`), sell (`#ef4444`), hold (`#6b7280`) |
| Bar widths | Proportional to `buy_count/total`, `sell_count/total`, `hold_count/total` |
| Unresolved segment | Transparent fill when `buy_count + sell_count + hold_count < total` -- no layout jump (D-05) |
| Y-axis labels | Bracket display names from `BRACKET_DISPLAY` (e.g., "Quants", "Doom-Posters") |
| Y-axis label style | 12px (`--font-size-label`), weight 400, `--color-text-secondary`, right-aligned, 80px reserved width |
| Bar corner radius | `rx="2"` for subtle rounding |
| Transition | Bar width changes animate over 600ms (`--duration-edge-fade`) via D3 transition (D-06) |
| Panel title | "Brackets" -- 18px (`--font-size-heading`), weight 600, `--color-accent`, bottom margin 8px |

**D3 pattern:** Follow `ForceGraph.vue` pattern -- `watch(() => snapshot.value.bracket_summaries, updateBars)` triggers D3 data join. No force simulation needed. Pure enter/update/exit on `<rect>` and `<text>` elements.

### RationaleFeed.vue (NEW)

**Purpose:** Scrolling list of recent agent rationale entries with animated arrivals and removals.

**Data source:** `inject('allRationales')` -- reads `allRationales: RationaleEntry[]` (newest first, capped at 20)

**Visual specification:**

| Element | Spec |
|---------|------|
| Container | `overflow-y: auto; flex: 1;` fills panel half height |
| Entry count | Maximum 20 visible entries (cap enforced in composable, D-08) |
| Entry layout | Flex row: agent name + signal chip on left, rationale text fills remaining width |
| Agent name | 14px (`--font-size-body`), weight 400, `--color-text-primary` -- format "Agent 07" |
| Signal chip | Inline badge: 12px (`--font-size-label`), weight 400, padding 2px 6px (`--space-xs`), border-radius 4px |
| Signal chip colors | `backgroundColor: SIGNAL_COLORS[signal] + '26'`, `color: SIGNAL_COLORS[signal]` (hold chip uses `--color-text-secondary` for text). Same pattern as `AgentSidebar.vue` lines 61-67. |
| Rationale text | 14px (`--font-size-body`), weight 400, `--color-text-muted`, line-height 1.5 |
| Text truncation | CSS `display: -webkit-box; -webkit-line-clamp: 2; -webkit-box-orient: vertical; overflow: hidden;` (D-10). Fallback: `max-height: calc(2 * 14px * 1.5); overflow: hidden;` for browsers without `-webkit-line-clamp` support. |
| Entry gap | 8px (`--space-sm`) between entries |
| Entry separator | No border between entries -- rely on spacing and background contrast |
| Panel title | "Rationale Feed" -- 18px (`--font-size-heading`), weight 600, `--color-accent`, bottom margin 8px |
| Agent name click | No link behavior -- agent name does NOT open sidebar (out of scope per CONTEXT.md) |

**Animation specification (D-11, D-12):**

| Transition | Property | Duration | Easing |
|------------|----------|----------|--------|
| Enter (slide-in) | `transform: translateY(-8px) -> translateY(0)` + `opacity: 0 -> 1` | 200ms (`--duration-feed-enter`) | `ease-out` (`--easing-enter`) |
| Leave (fade-out) | `opacity: 1 -> 0` | 200ms (`--duration-feed-exit`) | `ease-in` (`--easing-exit`) |
| Move (reorder) | `transform` | 200ms | `ease-out` |

Implementation: Vue `<TransitionGroup name="feed-entry">` wrapping `v-for` entries. CSS classes:

```css
.feed-entry-enter-active { transition: transform var(--duration-feed-enter) var(--easing-enter), opacity var(--duration-feed-enter) var(--easing-enter); }
.feed-entry-enter-from { transform: translateY(-8px); opacity: 0; }
.feed-entry-leave-active { transition: opacity var(--duration-feed-exit) var(--easing-exit); position: absolute; }
.feed-entry-leave-to { opacity: 0; }
.feed-entry-move { transition: transform var(--duration-feed-enter) var(--easing-enter); }
```

### useWebSocket.ts (EXTEND)

**New export:** `allRationales: Readonly<Ref<RationaleEntry[]>>`

**Behavior (D-08):**
- Maintain a `ref<RationaleEntry[]>([])` accumulator
- On each snapshot tick: prepend `snapshot.rationale_entries` to the front of the array
- Slice to 20 entries maximum: `allRationales.value = [...newEntries, ...existing].slice(0, 20)`
- Return as `readonly(allRationales)` alongside existing state

### App.vue (MODIFY)

**Changes:**
1. Destructure `allRationales` from `useWebSocket()` return (D-09)
2. Add `provide('allRationales', allRationales)` (D-09)
3. Restructure `.main-content` template: graph container + panel strip conditionally rendered (D-01)
4. Import and render `BracketPanel` and `RationaleFeed` inside panel strip

## Responsive Behavior

- Minimum viable viewport: 1024x768 (developer tool, not public-facing)
- Below 1024px width: panels stack vertically (bracket panel on top, feed below) instead of side-by-side -- each takes 50% of the 232px strip height
- No mobile breakpoint -- this is a desktop monitoring tool
