# Phase 10 -- UI Design Contract

> Visual and interaction contract for TUI Panels and Telemetry. Generated by gsd-ui-researcher. Extends the Phase 9 visual language with three new panels: rationale sidebar, telemetry footer, and bracket aggregation panel. Textual (Python terminal UI) -- not a web application.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Textual (>=8.1.1) |
| Preset | Custom dark theme `"alphaswarm"` via `textual.theme.Theme` (established Phase 9) |
| Component library | Textual built-in widgets (`Static`, `Container`, `Widget`, `RichLog`) + custom widgets |
| Icon library | Unicode symbols only (no icon library) |
| Font | Terminal monospace (user's terminal font, no override) |

**Note:** This is a Python terminal UI, not a web application. shadcn, Tailwind, and browser-based design systems do not apply. All styling uses Textual CSS (TCSS) and the Textual theme variable system. The `ALPHASWARM_THEME` registered in Phase 9 is reused without modification.

---

## Spacing Scale

Terminal UIs operate in character cells (1 unit = 1 character width or 1 line height). All spacing values are in cell units, not pixels. Inherits Phase 9 scale with additions for new panels.

| Token | Value | Usage |
|-------|-------|-------|
| none | 0 | Cell-to-cell adjacency in agent grid (unchanged) |
| xs | 1 | Inline padding within telemetry metric labels, rationale entry padding, bracket label padding |
| sm | 2 | Gap between telemetry metrics (pipe-separated), gap between bracket bar and percentage label |
| md | 3 | Padding inside sidebar container, padding inside bracket panel container |
| lg | 4 | Margin between main row and bottom row |

Exceptions:
- Agent grid cells remain exactly 3 characters wide by 1 line tall (unchanged from Phase 9).
- Rationale sidebar width: 40 characters (fixed). This accommodates `"> A_42 [BUY] momentum aligns with macro..."` format at ~50 character truncation plus signal brackets and agent ID.
- Telemetry footer height: 1 line (single horizontal metric row).
- Bracket panel row height: 1 line per bracket (10 rows total).
- Bottom row total height: 12 lines (1 telemetry + 10 bracket rows + 1 container padding).

---

## Typography

Terminal UI is monospace. "Size" refers to visual emphasis via weight and style, not pixel size. Textual renders in the user's terminal font at the terminal's configured size. Inherits Phase 9 typography with new roles for Phase 10 widgets.

| Role | Style | Weight | Usage |
|------|-------|--------|-------|
| Body | normal | normal (400) | Rationale text, telemetry metric values, bracket percentage labels |
| Label | normal | normal (400) | Telemetry metric names ("RAM:", "TPS:", "Queue:", "Slots:"), bracket display names |
| Heading | normal | bold (700) | App title "AlphaSwarm" in header (unchanged), sidebar section title "Rationale" |
| Status indicator | normal | bold (700) | Active simulation phase label (unchanged) |
| Signal tag | normal | bold (700) | Signal word in rationale entries: `[BUY]`, `[SELL]`, `[HOLD]` |

Only 2 weights used: **normal** (400) for data values and labels, **bold** (700) for emphasis (titles, signal tags, active status). No italic, no underline.

| Signal | Base Hue (HSL) | Min Lightness | Max Lightness |
|--------|---------------|---------------|---------------|
| BUY (bullish) | H=120, S=60% | L=20% (conf=0.0) | L=50% (conf=1.0) |

### Bracket Bar Signal Colors (New)

Bracket progress bars mirror the agent grid color semantics per D-07:

| Dominant Signal | Bar Fill Color | Source |
|-----------------|---------------|--------|
| BUY-dominant | `$success` (`#66BB6A`) | Bracket has more BUY than SELL or HOLD |
| SELL-dominant | `$error` (`#EF5350`) | Bracket has more SELL than BUY or HOLD |
| HOLD-dominant | `$secondary` (`#78909C`) | Bracket has more HOLD than BUY or SELL |
| No data | `$secondary` (`#78909C`) dim | No decisions yet (simulation not started) |

Bar background (unfilled): `#333333` (same as PENDING cell, provides consistent "empty" visual).

### Telemetry Warning Colors (New)

| Condition | Color | Trigger |
|-----------|-------|---------|
| RAM normal (< 80%) | `$foreground` (`#E0E0E0`) | Default telemetry text color |
| RAM elevated (>= 80%) | `$warning` (`#FFA726`) | GovernorMetrics.memory_percent >= 80.0 |
| RAM critical (>= 90%) | `$error` (`#EF5350`) | GovernorMetrics.memory_percent >= 90.0 |

## Component Inventory

### AgentCell (Unchanged from Phase 9)

- Dimensions: 3 characters wide, 1 line tall
- Content: No text content, color-only display
- Background: Computed from signal + confidence per Phase 9 color table
- Interaction: None

### AgentGrid (Unchanged from Phase 9)

- Layout: 10 columns x 10 rows, no gaps
- Dimensions: 30 characters wide x 10 lines tall
- Background: `$surface` (`#1E1E1E`)

### HeaderBar (Unchanged from Phase 9)

- Layout: Single horizontal line, full terminal width
- Height: 1 line
- Content: `  AlphaSwarm  |  Round X/3  |  [dot] Status  |  HH:MM:SS  `

### RationaleSidebar (New -- TUI-03)

- **Widget type:** `RichLog` (Textual built-in) or custom `Widget` with prepend capability
- **Dimensions:** 40 characters wide, fills main row height (matches agent grid height)
- **Background:** `$panel` (`#252525`)
- **Border:** Left border only, 1 character, `$secondary` color (`#78909C`) for visual separation from grid
- **Padding:** 1 character horizontal, 0 vertical (md=3 would waste space at this width)
- **Title:** "Rationale" in bold `$accent` (`#4FC3F7`), top of widget, 1 line
- **Content:** Scrolling log of high-influence rationale entries
- **Entry format:** `> {agent_short_id} [{signal}] {rationale_truncated}`
  - `>` prefix: `$secondary` (`#78909C`)
  - `{agent_short_id}`: `$foreground` (`#E0E0E0`), e.g. `A_42`
  - `[BUY]`: bold `$success` (`#66BB6A`)
  - `[SELL]`: bold `$error` (`#EF5350`)
  - `[HOLD]`: bold `$secondary` (`#78909C`)
  - `{rationale_truncated}`: `$foreground` (`#E0E0E0`), truncated at 50 characters with ellipsis
- **Scroll direction:** Newest entries prepend at top per D-04. Older entries scroll down.
- **Max visible entries:** Determined by available terminal height minus title line. No pagination controls.
- **Drain rate:** Up to 5 entries per 200ms tick per TUI-03.
- **Empty state:** No entries visible. Title "Rationale" still rendered. No placeholder text needed (sidebar is visually consistent as empty dark panel).

### TelemetryFooter (New -- TUI-04)

- **Widget type:** `Static` (single-line content update)
- **Dimensions:** Full width of bottom-left area, 1 line tall
- **Background:** `$panel` (`#252525`)
- **Padding:** 0 vertical, 1 horizontal
- **Content format:** `  RAM: {pct}%  |  TPS: {tps}  |  Queue: {depth}  |  Slots: {slots}  `
  - Metric labels ("RAM:", "TPS:", "Queue:", "Slots:"): `$secondary` (`#78909C`)
  - Metric values: `$foreground` (`#E0E0E0`) by default
  - RAM value: `$warning` at >=80%, `$error` at >=90% (see Telemetry Warning Colors)
  - Separator pipes: `$secondary` (`#78909C`)
- **Data sources:**
  - `RAM: {pct}%` -- `GovernorMetrics.memory_percent`, formatted as integer (`:.0f`)
  - `TPS: {tps}` -- `StateSnapshot.tps`, formatted as one decimal (`:.1f`)
  - `Queue: {depth}` -- `GovernorMetrics.active_count` (inference slots currently in use)
  - `Slots: {slots}` -- `GovernorMetrics.current_slots` (total available slots)
- **Update frequency:** Same 200ms `_poll_snapshot()` timer as all other panels. Reads from `StateSnapshot.governor_metrics` and `StateSnapshot.tps`.
- **Idle state (no metrics yet):** `  RAM: --  |  TPS: --  |  Queue: --  |  Slots: --  ` (dashes in `$secondary`)

### BracketPanel (New -- TUI-05)

- **Widget type:** Custom `Widget` rendering 10 rows of progress bars via Rich markup
- **Dimensions:** Fills bottom-right area width, 10 lines tall (one row per bracket) plus 1-line container padding top
- **Background:** `$panel` (`#252525`)
- **Padding:** 1 character top, 1 character horizontal
- **Row format:** `{bracket_name:<14}  [{bar_fill}] {pct}%`
  - `{bracket_name}`: Left-aligned, 14 characters, `$secondary` (`#78909C`). Uses `BracketSummary.display_name`.
  - `[{bar_fill}]`: 20-character bar. Filled portion uses signal color (green/red/gray per bracket bar colors table). Unfilled portion: `#333333`.
  - `{pct}%`: 3-4 characters, `$foreground` (`#E0E0E0`). Shows dominant signal percentage.
- **Bar fill computation:**
  - Dominant signal = `max(buy_count, sell_count, hold_count)` from `BracketSummary`
  - `dominant_pct = dominant_count / total * 100` (or 0 if total is 0)
  - Bar characters filled = `round(dominant_pct / 100 * 20)`
  - Fill character: Unicode full block `\u2588`
  - Empty character: Unicode light shade `\u2591`
- **Bar color:**
  - BUY-dominant: `$success` (`#66BB6A`)
  - SELL-dominant: `$error` (`#EF5350`)
  - HOLD-dominant: `$secondary` (`#78909C`)
  - Tie-breaking: BUY > SELL > HOLD (if counts are equal, the leftmost in this order wins)
- **Bracket order:** Follows `BracketConfig` list order from settings (Quants, Degens, Sovereigns, Macro, Suits, Insiders, Agents, Doom-Posters, Policy Wonks, Whales)
- **Update trigger:** On round completion only, via `StateSnapshot.bracket_summaries`. Not updated per-agent.
- **Idle state (no summaries yet):** All 10 bracket names shown with empty bars (all `\u2591`) and `0%` labels.

## Interaction Contract

### Snapshot-Based Rendering (Extended from Phase 9)

- Timer: `set_interval(1/5)` -- fires every 200ms (unchanged)
- On tick, `_poll_snapshot()` performs all of:
  1. Read `StateStore.snapshot()` for immutable `StateSnapshot`
  2. Diff agent states, update changed `AgentCell` backgrounds (unchanged)
  3. Update `HeaderBar` if phase/round/elapsed changed (unchanged)
  4. **New:** Drain up to 5 rationale entries from `StateSnapshot.rationale_entries` queue and prepend to `RationaleSidebar`
  5. **New:** Update `TelemetryFooter` from `StateSnapshot.governor_metrics` and `StateSnapshot.tps`
  6. **New:** Update `BracketPanel` from `StateSnapshot.bracket_summaries` (only re-render if summaries changed from previous snapshot)

### Rationale Queue Contract (TUI-03)

- **Producer:** `simulation.py` pushes entries to `StateStore.push_rationale()` after each round completes. Entries are selected by querying the top-N influence weight holders from the round's decisions.
- **Consumer:** `_poll_snapshot()` drains up to 5 entries per tick from the queue.
- **Queue type:** `asyncio.Queue` with `maxsize=50` (prevents unbounded growth if TUI is slow).
- **Entry dataclass:**
  ```
  @dataclass(frozen=True)
  class RationaleEntry:
      agent_id: str        # e.g. "A_42"
      signal: SignalType    # BUY, SELL, HOLD
      rationale: str       # truncated at 50 chars by producer
      round_num: int       # which round this came from
  ```
- **Drain method:** `StateSnapshot` exposes `rationale_entries: tuple[RationaleEntry, ...]` -- a snapshot of up to 5 entries drained from the queue at snapshot creation time.

### TPS Tracking Contract (TUI-04, D-05)

- **Producer:** `OllamaClient` extracts `eval_count` and `eval_duration` from each `ChatResponse` and calls `StateStore.update_tps(eval_count, eval_duration_ns)`.
- **Accumulation:** `StateStore` tracks `_cumulative_tokens: int` and `_cumulative_eval_ns: int`. `tps` = `_cumulative_tokens / (_cumulative_eval_ns / 1e9)` or 0.0 if no data.
- **Snapshot field:** `StateSnapshot.tps: float` -- running tokens-per-second.

### Simulation Lifecycle (Unchanged)

- TUI app owns the event loop
- Simulation runs as a `Textual Worker` (background asyncio task)
- StateStore is the shared bridge: simulation writes, TUI reads
- No IPC, no separate processes

### User Input (Unchanged)

- **Keyboard:** `q` or `Ctrl+C` to quit
- **Mouse:** None (no clickable elements)
- **Resize:** Textual handles terminal resize natively. Grid re-centers; sidebar and bottom row flex.

## State Mapping

### StateSnapshot Fields (Phase 10 Expansion)

The `StateSnapshot` frozen dataclass expands with three new fields:

| Field | Type | Purpose | Source |
|-------|------|---------|--------|
| `phase` | `SimulationPhase` | Header status label | Existing |
| `round_num` | `int` | Header round counter | Existing |
| `agent_count` | `int` | Grid size validation | Existing |
| `agent_states` | `dict[str, AgentState]` | Per-agent grid rendering | Existing |
| `elapsed_seconds` | `float` | Header elapsed time | Existing |
| `governor_metrics` | `GovernorMetrics \| None` | Telemetry footer: RAM, queue, slots | Existing |
| `tps` | `float` | **New:** Telemetry footer: tokens-per-second | `StateStore._cumulative_tokens / _cumulative_eval_ns` |
| `rationale_entries` | `tuple[RationaleEntry, ...]` | **New:** Rationale sidebar: up to 5 drained entries | `StateStore._rationale_queue` drain |
| `bracket_summaries` | `tuple[BracketSummary, ...]` | **New:** Bracket panel: per-bracket signal distribution | `simulation.py` post-round callback |

### StateStore New Methods

| Method | Signature | Purpose |
|--------|-----------|---------|
| `push_rationale` | `async (entry: RationaleEntry) -> None` | Push rationale entry to asyncio.Queue |
| `update_tps` | `(eval_count: int, eval_duration_ns: int) -> None` | Accumulate TPS from Ollama response metadata |
| `set_bracket_summaries` | `async (summaries: tuple[BracketSummary, ...]) -> None` | Store bracket summaries after each round |

### RationaleEntry (New Frozen Dataclass)

| Field | Type | Purpose |
|-------|------|---------|
| `agent_id` | `str` | Agent short ID for display (e.g. "A_42") |
| `signal` | `SignalType` | Signal for color-coding the `[BUY]`/`[SELL]`/`[HOLD]` tag |
| `rationale` | `str` | Truncated rationale text (max 50 characters, truncated by producer) |
| `round_num` | `int` | Round number for context |

## Accessibility Notes

- Color is the primary encoding for agent grid cells (unchanged from Phase 9).
- Rationale sidebar adds text-based signal information (`[BUY]`, `[SELL]`, `[HOLD]`) providing non-color signal identification.
- Telemetry footer is fully text-based with labeled metrics -- no color-only encoding.
- Bracket panel uses color for bar fills but also includes percentage labels for non-color access.
- Terminal screen readers can read all text content in sidebar, footer, and bracket panel.
- High contrast maintained: `$panel` (`#252525`, L~15%) background with `$foreground` (`#E0E0E0`, L~88%) text exceeds 4.5:1 luminance ratio.
- Warning/error colors (`$warning` `#FFA726`, `$error` `#EF5350`) on `$panel` background exceed 3:1 luminance ratio.
