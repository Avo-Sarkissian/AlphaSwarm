<objective>
Build the report viewer backend: a new FastAPI route file with two endpoints (GET read-through + POST background-task trigger), tests, and lifespan wiring. The POST handler mirrors cli.py `_handle_report` verbatim — load orchestrator, run ReportEngine, assemble, write_report, write_sentinel, unload — all inside an asyncio.create_task() for non-blocking 202 Accepted semantics. Phase guard (409), in-progress guard (409), services guard (503), and path-traversal guard (400) are all enforced.

**Revision round 1 (reviews mode):** Two MEDIUM concerns from the Codex cross-AI review are addressed:
1. **Async blocking I/O:** Replace synchronous `Path.exists()` / `Path.stat()` in the GET endpoint with `aiofiles.os.path.exists` / `aiofiles.os.stat` so the event loop is never blocked (CLAUDE.md Hard Constraint 1).
2. **Background task error surfacing:** Attach an `asyncio.Task.add_done_callback` to the generation task. The callback stores any exception into `app.state.report_generation_error[cycle_id]`. The GET endpoint reads from this dict and returns HTTP 500 with a machine-parseable error payload when the task failed, so the frontend stops polling immediately instead of timing out after 10 minutes.

**Also documented as accepted risk (LOW, both reviewers):** the atomic file-write race on `write_report`. Window is milliseconds; poll cadence is 3s. Noted in the threat model; no code change.

Purpose: Provides the REST API that the Vue ReportViewer modal (Plan 02) will call. GET is the polling target; POST spawns generation and returns immediately. Zero changes to src/alphaswarm/report.py — all Phase 15 logic is reused unmodified.

### aiofiles.os async filesystem API (Codex MEDIUM fix)

`aiofiles` is already a project dependency (declared in `pyproject.toml`; used by `src/alphaswarm/report.py`).
Its `aiofiles.os` submodule mirrors `os` / `os.path` with coroutine equivalents that wrap calls in
`asyncio.to_thread`-style offloading:

```python
import aiofiles.os

# Async versions of the stdlib calls (return coroutines):
exists = await aiofiles.os.path.exists(path)   # bool
statres = await aiofiles.os.stat(path)         # os.stat_result (has st_mtime)
```

`path` may be `str` or `Path`. Return types match the synchronous stdlib. These coroutines do NOT hold
the event loop. Reference: aiofiles README — https://github.com/Tinche/aiofiles#asynchronous-interface-to-osstat

       def _on_report_task_done(task: "asyncio.Task[Any]", cycle_id: str, app: FastAPI) -> None:
           """Done-callback for the background generation task.

           On failure: records {error, message} into app.state.report_generation_error[cycle_id]
           so the GET endpoint can surface 500 instead of the frontend polling to timeout.

           On success: no-op (file on disk is the success signal for GET).

           Must be safe to call on a CancelledError (treat as failure with message 'cancelled').
           """
           if task.cancelled():
               try:
                   errors = getattr(app.state, "report_generation_error", None)
                   if errors is None:
                       app.state.report_generation_error = {}
                       errors = app.state.report_generation_error
                   errors[cycle_id] = {"error": "report_generation_failed", "message": "cancelled"}
                   log.warning("report_task_cancelled", cycle_id=cycle_id)
               except Exception:  # defensive — callback must never raise
                   pass
               return

           exc = task.exception()
           if exc is None:
               # Success — leave app.state.report_generation_error untouched for this cycle.
               log.info("report_task_completed", cycle_id=cycle_id)
               return

           # Failure — record a machine-parseable error for the GET endpoint to surface.
           try:
               errors = getattr(app.state, "report_generation_error", None)
               if errors is None:
                   app.state.report_generation_error = {}
                   errors = app.state.report_generation_error
               errors[cycle_id] = {
                   "error": "report_generation_failed",
                   "message": str(exc) or exc.__class__.__name__,
               }
               log.error(
                   "report_task_exception_recorded",
                   cycle_id=cycle_id,
                   error=str(exc),
                   exc_type=exc.__class__.__name__,
               )
           except Exception:  # defensive — callback must never raise
               log.error("report_task_exception_record_failed", cycle_id=cycle_id)
