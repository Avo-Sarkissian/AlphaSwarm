# Phase 41.6 Plan 02: UI Revamp Wave 2 — Dashboard Surface Ports + Live Wiring Summary

**One-liner:** Ported the 6 dashboard surfaces (icons / viz / panels / states / tweaks / app_v2) from `AlphaSwarm-2/src/*.jsx` to `frontend/src/components/` as native ES modules per D-03, replaced the W1 placeholder app_v2 with a full live-wired shell consuming all 5 context hooks (Agents/Brackets/Telemetry/Rationales/Connection) plus simStart/simStop, reconciled the stylesheet via class-inventory diff (lifted 31 state-overlay rules from the archive, defined 3 W2-only classes), and hit all four production gates green (TS check, Vite build, no globals in src/, no mock leaks in dist/) — pending human visual verification at Task 5.

## Scope

Plan 41.6-02 implements **Wave 2 of 4** for the AlphaSwarm-2 Quant Terminal port. After this wave the main dashboard surfaces are live: the force graph renders 100 agents from `useAgents()`, the KPI strip ticks from `useTelemetry()` at backend cadence, the BracketList renders bracket summaries from `useBrackets()`, the RationaleFeed prints rationale events from `useRationales()`, lifecycle states (Idle / Seeding / MemoryPaused / Error) toggle on phase + connection signals, and the Topbar Run/Stop buttons fire `simStart(seed)` / `simStop()`.

Modal mounts (Interview, Report, Shock, Replay, CyclePicker, AdvisoryV2, DataSources, History, Settings) and the SignalWire ticker render as lightweight stub overlays so handlers wire end-to-end without crashing. W3 Plan 03 + W4 Plan 04 replace each stub.

### Lifecycle dispatch (priority order)

```text
useConnection().reconnectFailed → ErrorState
telemetry.memMb >= 90           → MemoryPausedState (pct)
telemetry.phase ∈ initializing|seeding → SeedingState (seed)
telemetry.phase == 'idle' && !running  → IdleState (seed, setSeed, onStart=simStart)
otherwise                       → running dashboard (Viz + KpiStrip + ConsensusRing + BracketList + RationaleFeed + Tooltip + Legend)
```

### Context-shape adapters at app_v2 boundary

The lifted contexts emit shapes that don't exactly match what the ported panels expect. Two adapters bridge the gap inside `app_v2.tsx`:

```typescript
// BracketList expects {bracket, display_name, total, buy_count, sell_count, hold_count}
// BracketSummaryView ships {bracket, display, total, buy, sell, hold}
const summaries = useMemo(() => brackets.map(b => ({
  bracket: b.bracket, display_name: b.display, total: b.total,
  buy_count: b.buy, sell_count: b.sell, hold_count: b.hold,
})), [brackets]);

// RationaleFeed expects {agent, bracket, signal, round, text, cites}
// RationaleView ships {agentId, round, text, citations, sources, ts}
const feedRationales = useMemo(() => rationales.map(r => {
  const a = agents.find(ag => ag.id === r.agentId);
  return {
    agent: r.agentId,
    bracket: a?.bracketDisplay ?? '',
    signal: a?.signal ?? 'hold',
    round: r.round,
    text: r.text,
    cites: r.citations ?? [],   // KR-41.1-10: stubbed [] until backend ships cited_agents
  };
}), [rationales, agents]);
```

Keeping the panels byte-identical to `AlphaSwarm-2/src/panels.jsx` makes future re-syncs from design easy. The adapter cost is two `useMemo`s in app_v2.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 — Bug] Pitfall #1 grep gate false-trigger on stale port comments**

- **Found during:** Task 3 verify
- **Issue:** My initial port of `app_v2.tsx` had a wiring-map comment block listing `window.AS_DATA.buildAgents`, `window.AS_DATA.bracketSummaries`, `window.AS_DATA.RATIONALES` literally to document what was being replaced. The negated grep `! grep -E 'window\.AS_DATA' frontend/src/components/app_v2.tsx` does not distinguish comments from code, so the verify failed.
- **Fix:** Rewrote the comment lines to describe the dropped surfaces semantically ("source's design-mock agent builder" / "source's mock summariser" / "source's static fixture") without the literal `window.AS_DATA` string.
- **Files modified:** `frontend/src/components/app_v2.tsx`
- **Commit:** `1f79354`

**2. [Rule 2 — Critical] Pitfall #3 advisory-literal grep failed on default seed**

- **Found during:** Task 4 production gate run
- **Issue:** I initially ported the source's `useState('Apple acquiring Anthropic for $500B — compute commitments unclear')` default verbatim. Pitfall #3 (W4 advisory-literal grep, also enforced in W2 task 4 acceptance) flags the literal `Apple acquiring Anthropic` and `$500B` as design-mock leakage.
- **Fix:** Changed default to empty string `useState<string>('')` — the user supplies their own seed via the Topbar input, which is the canonical entry per CONTRACT.md. The Onboarding flow in W4 also routes the user's seed back through `setSeed` before the dashboard mounts.
- **Files modified:** `frontend/src/components/app_v2.tsx`
- **Commit:** `46010ac`

**3. [Rule 3 — Blocking] Initial inventory diff polluted by ripgrep --help output**

- **Found during:** Task 4 PART A first run
- **Issue:** My first `rg -oh '...' frontend/src/components/` invocation expanded into argument parsing edge cases that dumped ripgrep's `--help` text into the "classes used" set, returning 462 spurious tokens including all the `--option` flag names.
- **Fix:** Replaced `rg` with plain `grep -roh '...' frontend/src/components/` plus `sed -E` to extract class tokens cleanly. Re-ran inventory: 128 used, 131 defined (new), 34 missing. No deviation impact on the lifted CSS — the result was the same set of state-overlay classes from the archive — but the diff would have been noisy if I'd missed the issue.
