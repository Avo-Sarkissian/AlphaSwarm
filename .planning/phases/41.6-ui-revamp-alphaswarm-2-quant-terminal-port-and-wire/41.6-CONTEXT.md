# Phase 41.6: UI Revamp — Port AlphaSwarm-2 (Quant Terminal) over Current React Frontend and Wire All Surfaces to Live Data — Context

**Gathered:** 2026-05-08

**Source:** Direct conversation (in-session 2026-05-08)

<domain>
## Phase Boundary

**In scope:**
- Treat `AlphaSwarm-2/` (extracted from `AlphaSwarm-2.zip`) as the **new standard frontend**. Canonical entry: `v2.html` → `app_v2.jsx` (16 JSX files + `styles.css`, ~5,400 LOC, currently in React-from-CDN + Babel-standalone format).
- Archive current `frontend/` to `frontend-react-archive/` (mirrors how Phase 41.1 archived Vue → `frontend-vue-archive/`).
- Scaffold fresh `frontend/` from AlphaSwarm-2 sources, converting CDN/Babel-standalone JSX to native ES modules in a clean Vite + React 19 + TypeScript pipeline.
- Lift the wiring layer wholesale from the archive: `api/`, `context/`, `hooks/`, `adapter/`, `types.ts`, and `mocks/` (mocks scoped to dev-only per KR-41.1-02). `CONTRACT.md` is byte-identical between old and new — adapters, REST clients, and Context shapes carry over without schema work.
- Wire every surface to live data via the lifted wiring layer, surface by surface in waves.
- New parity matrix tailored to the new surfaces (~25–30 rows). Human UAT sign-off per row.
- Audit all 11 KR-41.1-NN known regressions for cheap closures; carry forward unclosed ones; file new KR-41.6-NN for AdvisoryV2 schema gaps and `new_features.jsx` deferral.

**Out of scope:**
- Backend WebSocket schema changes, broadcaster cadence, REST route shapes (CONTRACT.md is unchanged and binding).
- Backend extensions for `new_features.jsx` (What-If Compare, Watch Mode, Multi-seed Synthesis, Seed Suggestions) — these need run-pairs / scheduling / multi-seed engine that don't exist; deferred to a future phase.
- Re-derivation of decisions already locked in Phase 41.1 (stack, contract, adapter shapes) or Phase 41.2 (ReportModal real-wiring, AdvisoryModal `portfolio_outlook` + `items[]` rendering, per-cycle lock + 404 surfacing).
- Adding shares/basis/last to the backend `AdvisoryReport` for AdvisoryV2 (stub + KR instead).

</domain>

<decisions>
## Implementation Decisions

### Strategy
- **D-01 — Surface-by-surface waves.** Sequencing: (W1) scaffold + design tokens + shell, (W2) viz + panels + states, (W3) modals + v2 (incl. AdvisoryV2), (W4) 5 net-new files + UAT. Affords partial-progress visibility and per-surface rollback.
- **D-02 — AlphaSwarm-2 is the new standard frontend.** Archive current `frontend/` → `frontend-react-archive/`. Scaffold fresh `frontend/` from AlphaSwarm-2 sources. Lift wiring layer wholesale from archive. NOT a diff-merge of current; not a parallel-folder build. User intent: *"I want this to be treated as a separate and new front end. some of the wiring can be copied over, but this is the new standard."*
- **D-03 — Port format conversion.** Convert each `AlphaSwarm-2/src/*.jsx` from CDN-globals format (`const { useState } = React;`, `window.AS_DATA`) to native ES modules: explicit `import { useState } from 'react'`, named exports per component, file-level imports for cross-component dependencies. Keep `.jsx` extension during port; rename to `.tsx` is **Claude's discretion** (incremental fine).
- **D-04 — Breakage policy = stop and surface.** If a new design surface drops a feature the live wiring depended on, executor pauses and surfaces it for user decision (file as KR or accept the loss). No silent regressions, no "preserve current at the cost of design integrity."

### Wiring layer lift (from archive)
- **D-05 — Lift these directories wholesale, no edits beyond import-path updates:**
  - `frontend/src/api/` (advisory.ts, client.ts, holdings.ts, interview.ts, replay.ts, report.ts, simulation.ts) — REST clients
  - `frontend/src/context/` (AgentsContext, BracketContext, ConnectionContext, EdgesContext, RationalesContext, TelemetryContext) — 6 React Context providers
  - `frontend/src/hooks/` (useApi, useCurrentCycle, useEdges, usePolling, useWebSocket) — WebSocket + polling
  - `frontend/src/adapter/frame.ts` + `agentId.ts` — backend frame → CONTRACT shape adapters
  - `frontend/src/types.ts` — TS types matching CONTRACT.md
  - `frontend/src/mocks/` (wire.ts, sources.ts, holdings.ts) — KR-41.1-02 dev-only mocks for SignalWire + DataSources surfaces (production grep already enforced)
- **D-06 — Drop archive duplicates.** `frontend/src/types 2.ts`, `frontend/src/adapter/frame 2.ts`, `frontend/src/components/app.jsx` (legacy), and any `_smoke.tsx` artifacts do NOT lift. Source of truth is the un-suffixed file.
- **D-07 — App shell composition.** New `App.tsx` mirrors the current 6-context provider tree (ConnectionProvider → TelemetryProvider → AgentsProvider → BracketProvider → RationalesProvider → EdgesProvider → DashboardShell), with `DashboardShell` swapped from current `app_v2` to the ported AlphaSwarm-2 `app_v2`.

### AdvisoryV2 wiring (`AlphaSwarm-2/src/v2.jsx:285`)
- **D-08 — Mix: derive what's reasonable, stub the rest.** AdvisoryV2 is the new canonical advisory surface (replaces `AdvisoryModal` from Phase 41.1 + 41.2). Wire from backend `AdvisoryReport` (`src/alphaswarm/advisory/types.py`):
  - **Direct wire:** `consensus_signal`, `confidence`, `rationale_summary`, `position_exposure`, `portfolio_outlook`, `total_holdings`, `affected_holdings`
  - **Derive cheaply:** `weight = position_exposure / total_position_exposure`; `value = position_exposure` (or current value if available); `agents` = count of citations in `rationale_summary` (best-effort regex match on agent IDs like `Q-03`, `D-12`)
  - **Best-effort parse from `portfolio_outlook` markdown:** `rec` (primary recommendation headline/first H2), `risk` and `upside` (sections matching `## Risk` / `## Upside` if present)
  - **Stub + file as new KR-41.6-NN:** `shares`, `basis`, `last`, `sentiment` (no backend source). Render as `—` in UI.
- **D-09 — Render `portfolio_outlook` markdown via `renderMarkdown()`** in the `.adv-rec-body` container. Headline area: generic label (e.g., "Portfolio Outlook") or first heading from the markdown. Hardcoded "Trim AAPL 15–25%..." in the design is filler; do not preserve.
- **D-10 — All 4 tabs (Overview / Holdings / Risk / Isolation) port + wire** under the same "derive reasonable, stub rest" rule. Holdings tab and Overview's quick-holdings list use the same `items[]` data — single source of truth. Isolation tab is informational (no backend data needed).

### 5 net-new files
- **D-11 — Port + wire `interview_v2.jsx`** as the canonical interview surface (replaces InterviewModal in modals). Wire to `/api/interview/{agent_id}` via existing `frontend/src/api/interview.ts`. PERSONA hardcoded responses are filler — replace with the live response. KR-41.1-11 (non-streaming) carries forward.
- **D-12 — Port + wire `onboarding.jsx`** as the first-run flow. Wire Ollama check to `/api/health` (or equivalent); seed-entry "Run" button to `/api/simulation/start`. First-run gate: localStorage flag (e.g., `as_onboarding_complete`) — Claude's discretion on flag name.
- **D-13 — Port + wire `bracket_deep.jsx`** as full-screen bracket internals view. Wire to `useAgents()` filtered by bracket via existing AgentsContext. Mounts on bracket-row click in panels (per AlphaSwarm-2 design).
- **D-14 — `data.jsx` is reference-only.** Lift only the `BRACKETS` constant if needed for non-running states (idle/seeding overlays). Drop `mulberry32`/`buildAgents`/`bracketSummaries` — live data via `useAgents()` + `useBrackets()` replaces them.
- **D-15 — Defer `new_features.jsx` to a future phase.** No backend support exists for What-If Compare / Watch Mode / Multi-seed Synthesis / Seed Suggestions. File as new KR-41.6-NN. The `v3.html` entry already excludes it — confirms it's intentionally separable.

### Verification + Known Regressions
- **D-16 — Build a NEW parity matrix tailored to the new surfaces** (~25–30 rows). One row per surface: force graph, KPI strip, brackets panel, signal wire, rationale feed, interview_v2 (new), cycle history, report modal, AdvisoryV2 (4 tabs = 4 rows), data sources modal, settings, onboarding (new), bracket_deep (new), lifecycle states (idle/seeding/live/mempause/error/done). Each row: expected behavior + data source + KR ref if accepted. Human UAT sign-off per row.
- **D-17 — Audit all 11 KR-41.1-NN; close cheap ones.** For each KR, verify whether the underlying gap has actually been closed by intermediate work (Phase 41.2 advisory data plane fixes, Phase 41.4 cited_agents/agent_id adapter improvements, Phase 999.3 telemetry restoration). Close anything cheap; carry forward what hasn't moved. Document per-KR status in PLAN.md.
- **D-18 — File new KRs:**
  - **KR-41.6-NN (AdvisoryV2 stubs):** `shares`, `basis`, `last`, `sentiment` rendered as `—` (no backend source).
  - **KR-41.6-NN (`new_features.jsx` deferred):** What-If Compare / Watch Mode / Multi-seed Synthesis / Seed Suggestions not ported in 41.6.
  - **KR-41.6-NN (PERSONA filler in interview_v2):** Bracket-aware hardcoded persona prompts/responses removed; live `/api/interview` response is the canonical render.
  - **KR-41.6-NN (Onboarding Ollama check):** If `/api/health` doesn't expose Ollama state granularly, onboarding step 2 shows a coarse OK/FAIL indicator instead of the design's per-model status.

### Claude's Discretion
- CSS reconciliation: take AlphaSwarm-2's 536-LOC `styles.css` as the foundation; merge in only the design tokens / overrides from current 1417-LOC styles that are demonstrably needed for live data rendering (e.g., new color tokens added during 41.2/41.4 wiring). Do NOT carry the whole current stylesheet over wholesale — it will conflict with new design tokens.
- `.jsx` → `.tsx` conversion timing: incremental during the port is fine; full conversion not required for 41.6.
- Mocks fate (`mocks/wire.ts`, `mocks/sources.ts`, `mocks/holdings.ts`): lift as-is per D-05, behind dev-only import gate. Production bundle grep (per Plan 41.1-05) re-runs in 41.6 UAT.
- Onboarding localStorage flag name and gating logic.
- Exact regex pattern for citation agent ID extraction in AdvisoryV2 `agents` derivation (D-08).
- Whether `bracket_deep` mounts as a modal or replaces the dashboard (per AlphaSwarm-2 visual spec).

- **REST polling for async cycle work**: `report` and `advisory` POST endpoints return 202 + `task_id`; modal polls `/api/{report|advisory}/{cycle_id}` until `status === 'ok'` or timeout. AdvisoryV2 inherits this pattern.
