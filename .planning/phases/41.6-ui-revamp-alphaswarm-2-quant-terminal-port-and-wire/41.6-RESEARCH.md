## Summary

This is a **port + wire phase, not a backend phase**. CONTRACT.md is byte-identical between the archived `Claude Design UI/AlphaSwarm/` and `AlphaSwarm-2/`, so the existing 6-context provider tree, REST clients, WebSocket hook, frame adapter, and dev-only mocks lift wholesale per D-05. The real engineering is in three buckets:

1. **Mechanical conversion** — 16 `.jsx` files in `AlphaSwarm-2/src/` use `const { useState } = React;` destructuring, IIFE-style top-level code, and `Object.assign(window, {...})` exports. Every cross-file reference goes through `window.*`. Conversion is per-file: replace destructure with `import { useState } from 'react'`, replace `window.AS_DATA.X` with named imports from a new `src/data.ts` module (lifting only the `BRACKETS` constant per D-14), replace `window.Icon` with `import { Icon } from './icons'`, drop the `Object.assign(window, ...)` lines and rely on `export function` declarations. **Critical hoisting trap:** `v2.jsx:557` runs `Object.assign(window, { ... ReportModalV2 ... })` BEFORE the `function ReportModalV2(...)` declaration at line 565. This works in CDN globals (function hoisting) but is a runtime ReferenceError under strict ES modules — order of function declarations in the source file matters during port.

2. **Surface-by-surface wiring** — every `window.AS_DATA.X` consumer is replaced by a context hook (`useAgents`, `useBrackets`, `useTelemetry`, `useRationales`, `useEdgesCtx`, `useConnection`) or a REST call from `frontend/src/api/*`. The signal-wire and data-sources surfaces stay on dev-only mocks (KR-41.1-02 carries forward). New surfaces (`interview_v2`, `onboarding`, `bracket_deep`) wire to existing endpoints with one new KR for Ollama health granularity (D-18).

3. **AdvisoryV2 schema bridge** — the design's `holdingAnalysis` shape has 4 fields (`shares`, `basis`, `last`, `sentiment`) that the backend `AdvisoryReport` does NOT carry. Per D-08 the plan: direct-wire what exists, derive `weight`/`value`, parse rec/risk/upside from `portfolio_outlook` markdown, render the 4 missing fields as `—` with new KR-41.6-NN.

**KR-41.1 audit verdict (D-17):** Of 11 KRs, **2 are partially closed by intermediate work** (KR-41.1-04 memory% now overlayed live by broadcaster per Phase 999.3; KR-41.1-10 citations have a backend prompt nudge but the `RationaleEntry` dataclass still doesn't carry `cited_agents` to the wire — INFLUENCED_BY edges land in Neo4j and are queryable via `/api/edges/{cycle_id}?round=N`). The other 9 carry forward unchanged into 41.6.

**Primary recommendation:** Do not attempt automated codemod. The 16 files are small (~5,400 LOC total) and each has 2-6 `window.*` references — manual per-file conversion in W1+W2+W3 waves is faster and safer than building a transform. Lift the wiring layer first (W1), convert the chrome (App, viz, panels) second (W2), then modals + AdvisoryV2 + 5 net-new (W3+W4).

### Pattern 1: CDN-globals → ES Modules conversion (D-03)

**What:** Rewrite each `AlphaSwarm-2/src/*.jsx` from React-CDN format to native ES modules.

**Per-file transform recipe:**

```jsx
// BEFORE (CDN globals — what AlphaSwarm-2 ships):
const { useState, useEffect, useMemo, useRef } = React;        // line 1-ish

function ComponentA() {
  const { HOLDINGS } = window.AS_DATA;                          // global data
  return <Tooltip />                                             // global component
}

window.ComponentA = ComponentA;                                  // global export
// or:
Object.assign(window, { ComponentA, ComponentB });

// AFTER (ES modules):
import { useState, useEffect, useMemo, useRef } from 'react';
import { Icon } from './icons';                                  // for window.Icon consumers
import { useAgents } from '../context/AgentsContext';            // replace window.AS_DATA.buildAgents
import { fetchHoldings } from '../api/holdings';                 // replace window.AS_DATA.HOLDINGS
import { Tooltip } from './app_v2';                              // co-located in same file in source — split out if cross-file

export function ComponentA() {
  const holdings = useHoldings(); // hook around fetchHoldings
  return <Tooltip />
}
```

**Critical hoisting trap:** `v2.jsx:557-561` runs:
```js
Object.assign(window, {
  SignalWire, ModelStatus, DataSourcesTakeover,
  AdvisoryV2, AdvStat, AdvHoldingRow,
  ReportModalV2, ConvergenceFlow, InfluenceChart, Moment, Followup
});
```
But `function ReportModalV2(...)` is declared AT LINE 565 (after the assign). In CDN mode this works because `function` declarations hoist to the top of the module scope. **In strict ES module mode, `Object.assign` runs at module init before later declarations execute** — but since we are dropping `Object.assign(window, ...)` entirely and using `export function`, the issue dissolves. However: be careful when reading the file top-to-bottom to extract export lists; some functions appear AFTER their first usage in the source. Lift the full export set by grepping `^function `.

### `portfolio_outlook` markdown parser (D-09 + D-08 best-effort)

The backend writes `portfolio_outlook` as free markdown — there is no enforced section structure (verified by reading `advisory/types.py` — `portfolio_outlook: str` with no schema). So the parser is **best-effort heuristic**, not contract-enforced. Treat misses as `null` and fall back to derived/stubbed values.

**Top-level extraction (for the Overview "PRIMARY RECOMMENDATION" block):**

```typescript
// src/lib/advisoryParse.ts
export interface ParsedOutlook {
  headline: string | null;     // first H1/H2 heading text
  summary: string;              // remainder rendered as markdown
}

export function parseOutlook(md: string): ParsedOutlook {
  const headingMatch = md.match(/^\s*#{1,2}\s+(.+?)\s*$/m);
  const headline = headingMatch ? headingMatch[1].trim() : null;
  return { headline, summary: md };  // body is rendered through renderMarkdown() unchanged
}
```

**Per-ticker extraction (for `rec`/`risk`/`upside` per item) — informal:**

```typescript
// Ticker-section regex: looks for "## TICKER" or "### TICKER" then captures until next H2/H3 or end-of-doc
export function extractTickerSection(md: string, ticker: string): string | null {
  const re = new RegExp(
    `(?:^|\\n)\\s*#{2,3}\\s+\\$?${ticker}\\b[^\\n]*\\n([\\s\\S]+?)(?=\\n\\s*#{2,3}\\s+|$)`,
    'i'
  );
  const m = md.match(re);
  return m ? m[1].trim() : null;
}

// Within a section, look for "Risk:" / "Upside:" / "Recommendation:" inline labels
export function extractField(section: string, label: 'risk' | 'upside' | 'rec'): string | null {
  const labels: Record<string, RegExp> = {
    risk: /(?:^|\n)\s*\*?\*?(?:Risk|RISK)\*?\*?\s*[:\-]\s*([^\n]+)/i,
    upside: /(?:^|\n)\s*\*?\*?(?:Upside|UPSIDE|Opportunity)\*?\*?\s*[:\-]\s*([^\n]+)/i,
    rec: /(?:^|\n)\s*\*?\*?(?:Recommendation|REC|Action)\*?\*?\s*[:\-]\s*([^\n]+)/i,
  };
  const m = section.match(labels[label]);
  return m ? m[1].trim() : null;
}
```

**Citation count regex (D-08 `agents` field — count of agent IDs in `rationale_summary`):**

```typescript
// Agent IDs follow the pattern: 1 letter prefix + dash + 2-digit number, e.g. Q-03, D-12, X-01
// Prefixes from data.jsx PREFIX const: Q, D, S, M, U, I, A, X, P, W
export const AGENT_ID_RE = /\b[QDSMUIAXPW]-\d{2}\b/g;

export function countAgentCitations(text: string): number {
  const matches = text.match(AGENT_ID_RE);
  if (!matches) return 0;
  return new Set(matches).size;  // unique agents only
}
```

## Common Pitfalls

### Pitfall 1: Function declaration order in v2.jsx
**What goes wrong:** `v2.jsx:557-561` references `ReportModalV2` in `Object.assign(window, {...})` but the `function ReportModalV2(...)` declaration is at line 565 (AFTER). Works in CDN globals (function hoisting), works in ES modules too (within a module scope, function declarations hoist), but if you accidentally split this file into two modules during port and break the hoisting boundary, you'll get `ReferenceError: ReportModalV2 is not defined`.
**How to avoid:** Keep all function declarations within a single module file OR re-order so declarations precede the (now-deleted) export block.

### Pitfall 2: `window.Icon` consumers everywhere
**What goes wrong:** ALL 12+ files reference `<Icon name="..." />` directly without import. Easy to miss one during port — TypeScript catches it as a missing identifier, but only after `tsc -b` runs.
**How to avoid:** Add `import { Icon } from './icons'` (or `'../components/icons'` from non-component sites) as the FIRST manual edit per file. Running `tsc` between each file conversion surfaces leftover gaps.

### Pitfall 3: AdvisoryV2 design literal "Trim AAPL 15-25%"
**What goes wrong:** D-09 explicitly forbids preserving this filler. But during W3, a quick "first pass" might leave it in to make the visual look right while wiring. UAT will then see the literal in production.
**How to avoid:** Before wave merge, grep for `"Trim AAPL"`, `"Apple acquiring Anthropic"`, `"$500B"`, `"C_2026_0418_A1"` in all ported files except `AlphaSwarm-2/` source mirrors. None should remain.

### Pitfall 4: Mock data leaking to production bundle
**What goes wrong:** `import { SIGNAL_WIRE_SEED } from '../mocks/wire'` at module top runs unconditionally. Vite includes it in the production bundle. KR-41.1-02 explicitly requires production bundle grep gate.
**How to avoid:** Either (a) gate at consumer with `import.meta.env.DEV ? (await import('../mocks/wire')).SIGNAL_WIRE_SEED : []`, or (b) rely on the existing pattern from Phase 41.1-05 (CONTEXT.md mentions "production grep already enforced"). Re-run grep gate post-build in 41.6 UAT.

### Pitfall 5: Polling timeout silent in usePolling
**What goes wrong:** Per Phase 41.2 D-06, `usePolling` was supposed to surface timeout errors. Reading current `frontend/src/hooks/usePolling.ts:54-66`, the timeout still silently `clearInterval(id)` and never sets `error` — the change appears NOT to have been completed in 41.2 (or was reverted). AdvisoryV2 / ReportModal will spin forever on backend hang.
**How to avoid:** Verify and patch `usePolling.ts:54-60` during the W3 wiring lift — set `setError(new Error('polling timed out after N attempts'))` when `n >= maxAttempts`. This is a pre-existing bug carry-forward that may need a quick task or a 41.6 micro-fix.

### Pitfall 6: Onboarding gate before wiring complete
**What goes wrong:** If Onboarding renders BEFORE the WebSocket subscribes, the dashboard transitions from Onboarding → AwaitingFrame → ConnectedFirstFrame. If the user clicks "Run" in Onboarding but the WebSocket isn't yet subscribed, the first frames are missed. (Probably OK because the simulation takes seconds to spawn, but worth documenting.)
**How to avoid:** Move `useWebSocket()` ABOVE the Onboarding gate in `App.tsx` — keep WS open the entire session, let Onboarding only block the dashboard render.

### Pitfall 7: D-08 regex false positives on agent IDs
**What goes wrong:** The agent ID regex `/\b[QDSMUIAXPW]-\d{2}\b/g` matches `Q-03` correctly but might also false-match arbitrary text like "M-12" (a date abbreviation? a measurement?) in `rationale_summary`. Bracket prefixes are short and could collide.
**How to avoid:** Acceptable for first cut — `agents` is a derived display number, not a contract. Document as KR-41.6-10 ("derived from regex, not contract"). If false positives are problematic in UAT, narrow the prefix set or look up against the live `useAgents()` agent ID list.

### Pitfall 8: React 18 vs 19 subtle behavior gaps
**What goes wrong:** AlphaSwarm-2 was authored against React 18.3.1 (per `v2.html:1976` UMD). Most React 19 changes are additive, but: `useMemo` with empty dep array runs once in 18, twice in 19 StrictMode (already true in 18 StrictMode). Some legacy `componentWillMount`-style patterns are removed. Unlikely to bite this codebase (function components only, no class components seen in any AlphaSwarm-2 file).
**How to avoid:** Run with React.StrictMode on (current `main.tsx:9-12` already does this). Watch console for "deprecated" warnings during W2/W3 UAT. None expected.

```tsx
              <div
                className="adv-rec-body"
                dangerouslySetInnerHTML={{ __html: outlookHtml }}
              />
            </div>
            <div className="adv-summary-stats">
              <AdvStat label="PORTFOLIO" value={`$${(holdingAnalysis.reduce((a,h) => a+h.value, 0)/1000).toFixed(0)}K`} sub={`${report?.total_holdings ?? 0} holdings`} />
              <AdvStat label="AFFECTED" value={`${report?.affected_holdings ?? 0}/${report?.total_holdings ?? 0}`} sub="of portfolio" tone="sell" />
              {/* ... */}
            </div>
            <div className="adv-quick-holdings">
              <div className="label">SWARM VIEW ON YOUR HOLDINGS</div>
              {holdingAnalysis.map(h => (
                <AdvHoldingRow key={h.ticker} h={h} totalValue={holdingAnalysis.reduce((a,x) => a+x.value, 0)} />
              ))}
            </div>
          </div>
        )}
        {/* tab === 'holdings', 'risk', 'isolation' similar; Risk tab is design placeholder per KR-41.6-05 */}
      </div>
    </div>
  );
}
```

   - What's unclear: Is the Compare feature important enough to render with placeholder copy, or should the "Compare" button be hidden entirely?
