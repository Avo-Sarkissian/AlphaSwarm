## Integration Gaps (Blocking)

### BUG-01 — Property name mismatch in `read_signal_flips` Cypher

**Files:** `src/alphaswarm/graph.py` lines 1356, 1359
**Affects:** GRAPH-02, REPORT-02

`_batch_write_episodes_tx` writes the round number as `round`:
```cypher
CREATE (re:RationaleEpisode { ..., round: $round_num, ... })
```

But `read_signal_flips` reads it back as `re.round_num`:
```cypher
re.round_num AS round_num,
ORDER BY re.round_num
```

`re.round_num` does not exist — this silently returns `null` for all rows. The signal flip analysis section of the report receives corrupt data.

**Fix:** Change `re.round_num` → `re.round` in `read_signal_flips` query.

---

### BUG-02 — ReACT system prompt tool names do not match CLI registry keys

**Files:** `src/alphaswarm/report.py` lines 32-38, `src/alphaswarm/cli.py` lines 703-712
**Affects:** REPORT-01, REPORT-02, REPORT-03

System prompt teaches LLM:
```
- consensus_summary
- signal_flips
```

CLI registry keys:
```python
"bracket_summary"        # LLM never calls this
"signal_flip_analysis"   # LLM never calls this
```

Two of 8 tools are unreachable by the LLM. The bracket consensus summary section and signal flip section will always be missing from generated reports.

**Fix (option A):** Rename CLI registry keys to match system prompt: `bracket_summary` → `consensus_summary`, `signal_flip_analysis` → `signal_flips`. Update `TOOL_TO_TEMPLATE` and `SECTION_ORDER` in sync.

**Fix (option B):** Update system prompt to use registry key names.
