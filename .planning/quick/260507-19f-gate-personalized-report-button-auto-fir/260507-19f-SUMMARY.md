#### Schema mismatch + fix

Writer (`alphaswarm.advisory`) emits `portfolio_outlook` as the markdown narrative field. Modal `useMemo` at `frontend/src/components/modals.jsx:381-389` only read `data.content || data.narrative`, so the rendered HTML was empty even though the JSON file existed on disk.

**Fix (frontend-only, 1-line extension):**

```diff
  const html = useMemo(() => {
    const content =
      (data &&
        (typeof data.content === 'string'
          ? data.content
          : typeof data.narrative === 'string'
          ? data.narrative
+         : typeof data.portfolio_outlook === 'string'
+         ? data.portfolio_outlook
          : '')) || '';
    return renderMarkdown(content);
  }, [data]);
```

Chose Option A (frontend extension) over Option B (rename writer field) because the writer schema is consumed by other downstream readers and re-running the sim to regenerate files was unnecessary ‚Äî existing `advisory/522c76b9-‚Ä¶json` validated the render after HMR.

**Smoke after fix:** Modal now renders the markdown narrative + PORTFOLIO CONTEXT block (AAPL/AMZN/ARM/‚Ä¶ holdings table) cleanly. Network tab GET-only confirmed.
