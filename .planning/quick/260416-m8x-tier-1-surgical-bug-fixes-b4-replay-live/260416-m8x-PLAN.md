<objective>
Apply five surgical Tier 1 bug fixes identified in `.planning/BUGFIX-CONTEXT.md`:

- **B4:** Prevent replay from starting while a live simulation is running, and prevent a live simulation from starting while a replay is active (bi-directional 409 guard).
- **B7:** Close the writer-side leak in `ConnectionManager._writer` so a crashed client that never routes through `disconnect()` no longer leaves stale entries in `_clients` / `_tasks`.
- **B8:** Eliminate the race between `SimulationManager._on_task_done` releasing the lock and the fire-and-forget `_reset_phase_to_idle()` task ‚Äî reset phase synchronously *before* the lock is released.
- **B9:** Serialize `ReplayManager.advance()` and `ReplayManager.stop()` behind `self._lock`, matching `start()`.
- **B10:** Remove the duplicate `set_phase(SimulationPhase.COMPLETE)` call at `simulation_manager.py:107`; `simulation.py:1109` is the single source of truth.

Purpose: Tighten lifecycle correctness around the live-simulation/replay boundary and the WebSocket client registry without introducing broader refactors or touching shock/bracket/replay-frontend code (those are separate phases).

Output: Five production-code fixes + one consolidated regression-test commit, preserving the async-first, non-blocking contract (no new locks on hot paths, no blocking I/O on the main loop, `structlog` for any new log events).
</objective>
