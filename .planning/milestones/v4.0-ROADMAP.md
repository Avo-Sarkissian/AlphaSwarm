# Milestone v4.0: Interactive Simulation & Analysis

**Status:** ✅ SHIPPED 2026-04-12
**Phases:** 24-28
**Total Plans:** 13

## Overview

Interactive simulation layer: exportable HTML reports with SVG charts and Schwab portfolio overlay, mid-simulation shock injection with governor suspend/resume, shock impact analysis with bracket delta mode, and full simulation replay from stored Neo4j state without re-running inference.

### Phase 26: Shock Injection Core

**Goal**: Mid-simulation shock injection with governor suspend/resume and ReplayStore isolation
**Depends on**: Phase 15
**Plans**: 5 plans

Plans:

- [x] 26-01: Wave 0 test scaffolding (20 failing RED stubs)
- [x] 26-02: `ResourceGovernor.suspend()` and `resume()` with memory-pressure guard
- [x] 26-03: `StateStore` shock bridge + `write_shock_event` Neo4j persistence
- [x] 26-04: `ShockInputScreen` modal overlay + `_poll_snapshot` edge-latch detection
- [x] 26-05: `_collect_inter_round_shock` simulation wiring (R1→R2 and R2→R3 gaps)

**Details:**
`ResourceGovernor.suspend()/resume()` enable mid-simulation pause; `resume()` memory-pressure guard lives at the callee to prevent TOCTOU races. `ShockInputScreen` modal allows operator to inject free-text shock event; `_shock_window_was_open` latch prevents re-pushing on consecutive `_poll_snapshot` ticks. `StateStore` shock bridge signals shock availability to `_collect_inter_round_shock`, which uses nested try/finally: inner for `close_shock_window`, outer for `governor.resume()`. `write_shock_event` persists `ShockEvent` node with `HAS_SHOCK` edge to the `Cycle` node. `SHOCK_TEXT_MAX_LEN = 4096` caps prompt injection size.

### Phase 28: Simulation Replay

**Goal**: Re-render a past simulation cycle from stored Neo4j state without re-running inference
**Depends on**: Phase 27
**Plans**: 3 plans

Plans:

- [x] 28-01: `SimulationPhase.REPLAY` enum + `ReplayStore` + 4 graph read methods + 14 Wave 0 tests
- [x] 28-02: `CyclePickerScreen` overlay + full TUI replay mode + `alphaswarm replay` CLI subcommand
- [x] 28-03: Human verification of all 7 replay scenarios

**Details:**
`SimulationPhase.REPLAY = "replay"` added to types.py. `ReplayStore` uses no-drain snapshot semantics (never queued) with round-filtered `agent_states`. Four parameterized Cypher read methods: `read_full_cycle_signals` (Decision-first, perf_counter timing), `read_completed_cycles` (Round-3 existence check), `read_bracket_narratives_for_round`, `read_rationale_entries_for_round`. `CyclePickerScreen` shows formatted cycle list; auto-selects single cycle, `OptionList` picker for multiple. TUI replay mode: amber REPLAY header badge, 3-second auto-advance timer, `p`/`space`/`right`/`escape` bindings, sidebar dedup via `_replay_last_rendered_round` latch, stale-load guard after each `await`. CLI `alphaswarm replay --cycle <id>` with `AppState(with_ollama=False)`. `_replay_store` set BEFORE phase change to prevent `_poll_snapshot` race.

## Milestone Summary

**Decimal Phases:**

- Phase 25: Schwab Portfolio Overlay (inserted between Phase 24 and 26 for brokerage integration)

**Key Decisions:**

- SVG charts via pygal (not Plotly) — HTML reports under 1MB vs 15MB+ (Phase 24)
- Schwab CSV loaded in-memory only — simulation stays uncontaminated; portfolio data never persisted to Neo4j (Phase 25)
- Governor suspend/resume guard lives at callee (governor.py) — prevents TOCTOU race vs caller-side check (Phase 26)
- `SHOCK_TEXT_MAX_LEN = 4096` caps prompt injection size across 100 agents (M1 memory constraint) (Phase 26)
- `_shock_window_was_open` edge latch prevents re-pushing on consecutive `_poll_snapshot` ticks (Phase 26)
- `comparable_agents` is Cypher inner-join denominator (not 100) — rate percentages reflect actual sample size (Phase 27)
- `_bracket_panel_delta_active` latch set INSIDE async worker AFTER `enable_delta_mode()` succeeds — transient Neo4j failure retries on next tick (Phase 27)
- `ReplayStore` separate from `StateStore` — destructive snapshot drain and timer corruption make reuse unsafe (Phase 28)
- `_replay_store` set BEFORE any phase change to prevent `_poll_snapshot` race (Phase 28)
- `key_escape` override instead of BINDINGS entry — avoids Textual BINDINGS conflict with Screen overlays (Phase 28)
- CLI replay handler is synchronous (like `_handle_tui`) — Textual owns the event loop (Phase 28)

**Issues Resolved:**

- Phase 26 worktree base predated Phase 26 `_shock_window_was_open` — auto-fixed in Plan 27-01
- Merge conflict in Phase 26 test dropped from Phase 27 worktree — restored in `fix(27)` commit
- `test_report.py` assembler failures (19 tests) across Phase 26-28 worktrees — pre-existing Phase 24/25 not yet merged; resolved at integration

**Issues Deferred:**

- `read_full_cycle_signals` Cypher performance for COLLECT aggregation across 600+ nodes — needs profiling under load (noted in Phase 28 state)
- `reset_delta_mode()` has no reset path for multiple simulation runs in the same TUI session — single-session limitation documented

**Technical Debt Incurred:**

- BracketPanel delta mode latch (`_bracket_panel_delta_active`) has no reset path across TUI session without restart — acceptable for single-session use
- Standalone argparse parser in CLI replay tests vs `main()` inline parser — no `_build_parser()` factory extracted (deferred)
