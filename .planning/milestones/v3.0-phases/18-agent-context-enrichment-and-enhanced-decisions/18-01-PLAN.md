## Resolved Review Issues in This Plan

**Blocker #3 (Codex): End-to-end ticker path**
CONFIRMED: `SeedEvent.tickers` is always empty. The orchestrator prompt in `seed.py` only extracts
entities (name/type/relevance/sentiment), not tickers. The seed parser `_try_parse_seed_json()`
constructs `SeedEvent(raw_rumor=..., entities=..., overall_sentiment=...)` without parsing tickers.
STATE.md explicitly notes: "SeedEvent.tickers is always empty until Phase 18 wires the ticker
extraction output into seed parsing". Task 1 below fixes this by updating the orchestrator prompt
to also extract tickers and updating the seed parser to populate SeedEvent.tickers.

**Blocker #2 (Codex): Macro-agent spec mismatch**
RESOLVED: CONTEXT.md D-04 is the locked user decision. It places Macro in the Earnings/Insider
slice (headlines + earnings surprise). The ROADMAP phrase "Macro agents see sector-level data"
is imprecise -- it describes the Earnings/Insider slice's macro-relevant content (earnings
surprises, EPS, market cap, news headlines) which IS the data Macro analysts use for sector
assessment. The locked D-04 decision governs. No contradiction exists.

**Blocker #4 (Codex): Parse robustness for malformed ticker_decisions**
Task 2 adds a lenient parse helper `_lenient_parse_ticker_decisions()` in parsing.py that
drops individual invalid TickerDecision entries while preserving the top-level AgentDecision.
Tests cover: malformed direction string, wrong type for expected_return_pct, invalid nested
object. In all cases the top-level decision is preserved, not collapsed to PARSE_ERROR.

**Improvement #8: direction field and parse_error**
The JSON_OUTPUT_INSTRUCTIONS explicitly lists `"direction": "buy"|"sell"|"hold"` (NOT
"parse_error") in the schema example. While TickerDecision.direction reuses SignalType
(which includes parse_error), the prompt never shows parse_error as a valid direction.
If the LLM emits parse_error as a direction, the lenient parser will drop that entry.

**Improvement #10: Lenient ticker_decisions parse path**
Implemented via _lenient_parse_ticker_decisions() in parsing.py, called from
parse_agent_decision's Tier 1 and Tier 2 paths when full model_validate_json fails.
