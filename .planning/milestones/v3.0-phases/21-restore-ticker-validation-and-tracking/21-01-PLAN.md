<threat_model>
Phase 21 restores two code paths that touch untrusted remote content and the local filesystem. Both paths existed in Phase 16 and are being restored verbatim — no new threat surfaces are introduced, but the threats are documented here for ASVS L1 compliance.

**Threat 1: SEC CDN network download (untrusted remote content)**
- Surface: `_download_sec_tickers()` fetches `https://www.sec.gov/files/company_tickers.json` via httpx.
- Risk: Malicious CDN response, oversized payload, or man-in-the-middle injection of invalid JSON.
- Mitigations in restored code:
  - Response is parsed as JSON via `json.loads()` — non-JSON payloads raise `json.JSONDecodeError` and are caught.
  - The result is iterated as a dict-of-dicts; only the `ticker` field (a string) is extracted — no eval, no exec.
  - `httpx.AsyncClient` enforces timeout via `httpx.TimeoutException`; ConnectError propagates to caller.
  - The file is written atomically via temp-file-rename — a truncated download does not corrupt an existing file.
- Residual risk: No content-length or hash validation against a known good manifest. Acceptable for ASVS L1 (SEC CDN is trusted authority).

**Threat 2: Atomic file write pattern**
- Surface: `_download_sec_tickers(dest)` writes to a temp file in `dest.parent`, then renames to `dest`.
- Risk: If the process is killed after the temp file is created but before rename, a stale temp file remains.
- Mitigations:
  - The `finally` block in the restored code calls `tmp.unlink(missing_ok=True)` on any exception before re-raising, ensuring cleanup even on error.
  - `Path.rename()` is atomic on POSIX filesystems (macOS/Linux) when src and dst are on the same mount.
- Residual risk: Non-POSIX or cross-device rename (not applicable on macOS M1 with default single-volume layout).

**Threat 3: User-Agent header in httpx request**
- Surface: `_download_sec_tickers()` sends `User-Agent: AlphaSwarm admin@alphaswarm.local`.
- Risk: The SEC CDN requires a descriptive User-Agent per their scraping policy; missing or bot-like headers result in 403 or rate-limit responses.
- Mitigation: The restored header matches Phase 16's original format which was verified working against the live SEC CDN during Phase 16 execution.
- Note: The email in the User-Agent is a placeholder (`admin@alphaswarm.local`) — not a real address, which is acceptable for local-only tooling.

**Threat 4: Module-level cache (`_ticker_set`) cross-request pollution**
- Surface: `_ticker_set: set[str] | None = None` is a process-lifetime global.
- Risk: In tests, one test that sets `_ticker_set` to a value will pollute subsequent tests.
- Mitigation: The `autouse` fixture `reset_ticker_cache` in `test_ticker_validator.py` resets `tv._ticker_set = None` before and after each test. This is mandatory and is verified by the test order in Wave 0.
</threat_model>
