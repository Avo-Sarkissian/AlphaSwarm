## Cross-Phase Integration Findings

### BREAKING: TICK-02 — SEC Ticker Validation Deleted

**Root cause:** Commit `7ba7efa` (`feat(17-01): install yfinance and define MarketDataSnapshot model`) deleted `ticker_validator.py`, both Phase 16 ticker test files, and removed all `ticker_validator` wiring from `seed.py` and `parsing.py`. This was a mass cleanup in what should have been a scoped Phase 17 commit.

**Impact:** Tickers extracted by the LLM are accepted unconditionally. Invalid symbols (misspellings, hallucinations, non-SEC-listed names) will flow into `fetch_market_data()` and produce degraded snapshots silently.

**Fix:** Restore `ticker_validator.py` from git history, restore `parse_seed_event()` kwarg, restore `dropped_tickers` field on `ParsedSeedResult`, restore CLI display.

### TECH DEBT: pyproject.toml Missing yfinance

`yfinance>=1.2.0` was in `pyproject.toml` after Phase 17 Plan 01 but was dropped when Plan 03 touched pyproject.toml. Fresh `uv sync` on another machine will ImportError.

**Fix:** Add `"yfinance>=1.2.0"` to `[project.dependencies]` in pyproject.toml.

### PRE-EXISTING: REACT_SYSTEM_PROMPT Tool Name Mismatch

`REACT_SYSTEM_PROMPT` in `report.py` references `consensus_summary` and `signal_flips` but the runtime tool registry uses `bracket_summary` and `signal_flip_analysis`. When the LLM follows the prompt, it calls unknown tool names and those sections are absent from the report. This predates v3.0 but degrades DRPT-01.
